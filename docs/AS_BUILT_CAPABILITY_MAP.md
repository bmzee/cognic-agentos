# AgentOS v1 — As-Built Capability Map

**Date:** 2026-06-12 · **Baseline:** `main @ 1af0e62` (post Sprint 13.5c3, PR #64) · **Status:** AUTHORITATIVE "what v1 is now" map
**Rule:** every claim below is grounded in repo evidence (source modules, ADR sections, tests, e2es). Nothing aspirational. Where a capability is named but unbuilt, its state says so plainly.
**Posture vocabulary:** `production-wired` (a deployed runtime reaches it through the composition root) · `seam-only` (built + test/DI-proven; no production caller constructs it yet) · `env-gated live proof` (real-infrastructure proof exists but is opt-in, run pre-merge by the operator) · `partial` · `missing` (named, sequenced, deliberately unscoped).

This is NOT a proof of concept. AgentOS v1 is a production-hardened, bank-deployable governance kernel + runtime: 128 modules ride the durable critical-controls coverage gate (95% line / 90% branch, re-verified on fresh `--cov-branch` data at every promotion), the full suite stands at ~10.4k tests / 0 failed, and CI runs live Postgres + Oracle integration lanes plus image-size budgets on every PR.

## Cross-cutting as-built facts

- **Critical-controls gate:** 129 modules at 95/90 floors (`tools/check_critical_coverage.py`), verified at promotion on fresh data every time (`core/emergency/quotas.py` promoted at Sprint 13.6b).
- **Evidence substrate:** hash-chained `decision_history` + `audit_event` (Doctrine Lock D atomicity; canonical form per `core/canonical.py` is the evidence-pack wire format, ADR-006); ISO 42001 evidence-pack export (Sprint 9).

- **Eval/quality gates:** bulk eval + LLM-judge + live replay + adversarial gate feeding the pack 5-gate composer (ADR-010/011; Sprints 12–13c).
- **Per-tenant config overlay:** tighten-only, fail-closed (ADR-023, Wave-2 scope shipped).
