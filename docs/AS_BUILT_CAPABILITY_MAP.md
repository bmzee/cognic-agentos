# AgentOS v1 ‚Äî As-Built Capability Map

**Date:** 2026-06-26 ¬∑ **Baseline:** `main @ 8ebf952` (post Proof 1b-2, PR #103) ¬∑ **Status:** AUTHORITATIVE "what v1 is now" map
**Rule:** every claim below is grounded in repo evidence (source modules, ADR sections, tests, e2es). Nothing aspirational. Where a capability is named but unbuilt, its state says so plainly.
**Posture vocabulary:** `production-wired` (a deployed runtime reaches it through the composition root) ¬∑ `seam-only` (built + test/DI-proven; no production caller constructs it yet) ¬∑ `env-gated live proof` (real-infrastructure proof exists but is opt-in, run pre-merge by the operator) ¬∑ `partial` ¬∑ `missing` (named, sequenced, deliberately unscoped).

This is NOT a proof of concept. AgentOS v1 is a production-hardened, bank-deployable governance kernel + runtime: 134 modules ride the durable critical-controls coverage gate (95% line / 90% branch, re-verified on fresh `--cov-branch` data at every promotion), the full suite stands at ~11k tests / 0 failed, and CI runs live Postgres + Oracle integration lanes plus image-size budgets on every PR. The governed pack loop is proven both **in-process** (Proof 1a, PR #96) and **deployed** (Proof 1b-2, PR #103 ‚Äî a real in-tree MCP tool pack invoked end-to-end through a deployed kernel on kind/Helm; tool invocation, not yet an LLM-agent loop or a separate-repo pack).
