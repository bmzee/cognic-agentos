## What ships in `feat/sprint-5-mcp-host` after Sprint 5

### Five new critical-controls modules

- **`src/cognic_agentos/protocol/mcp_authz.py`** — admission-side OAuth/PRM authz client. Three-path PRM discovery in priority order (`WWW-Authenticate: Bearer resource_metadata="..."` header → endpoint-specific `/.well-known/oauth-protected-resource<endpoint-path>` → root `/.well-known/oauth-protected-resource`); per-tenant authorization-server allow-list (`secret/cognic/<tenant>/mcp-as-allowlist`); RFC 8707 resource indicator on every token request; audience-validation rejection of mismatched `aud` claim; minimum-scope tokens per pack manifest; **`Token` dataclass with token-free invariant** — frozen+slotted prevents accidental `__dict__` exposure, custom `__repr__` redacts the bearer value (never logged, never in audit payloads). 13-value closed-enum `AuthzReason` literal split into 11 registration-boundary values mappable to `RefusalReason` and 2 runtime-only values (`mcp_step_up_unauthorised`, `mcp_authorisation_lost`) — split enforced by `_RUNTIME_ONLY_AUTHZ_REASONS` frozenset + drift detector in `test_refusal_reason_completeness.py`. Production-grade `httpx.AsyncClient` (no `mcp` SDK at admission per R3 P1 — admission-side is pure HTTP+OAuth, runtime-side uses the SDK). Per-file coverage: 99.29% line / 96.94% branch.
- **`src/cognic_agentos/protocol/mcp_capabilities.py`** — signed-manifest capability validator. **12-value** closed-enum `ValidationReason` covering the Sprint-5 capability + STDIO four-gate vocabulary: the original ten (`mcp_anonymous_refused`, `mcp_resources_declared_but_no_list`, `mcp_sampling_default_denied`, `mcp_elicitation_form_restricted_data_class`, `mcp_caching_ttl_restricted_data_class`, `mcp_stdio_manifest_incomplete`, `mcp_stdio_manifest_shell_metacharacter`, `mcp_stdio_command_not_allowlisted`, `mcp_stdio_disabled_in_sprint_5` Decision Lock umbrella, `mcp_transport_unsupported` from R1 P1 #2) plus the two T15-R1 / R2 hardening additions: `mcp_http_manifest_shape_invalid` (T15 R1 P2 #6 — HTTP-family `server_url` / `scopes` shape gate that fires before the auth probe) and `mcp_tool_data_classes_shape_invalid` (T15 R2 P2 — fail-closed shape gate on a tool's `data_classes` field that fires before gates 4 + 5 would consume the field). Sampling four-condition gate via OPA Rego (`policies/_default/sampling.rego` — pack-declared + tenant-permitted + cloud-policy-tier-consistent + cloud-policy-allow-external-llm-consistent). STDIO manifest precedence order: incomplete → shell-metacharacter → command-not-allowlisted → Decision Lock umbrella (umbrella fires LAST so operators see actionable diagnostics first). Pack-controlled-TOML safety (R2 P2): `_safe_get_dict` walks safely through non-dict intermediates so a pathological `{"tool": "bad"}` produces a closed-enum refusal rather than `AttributeError`. Per-file coverage: 100 / 100.
- **`src/cognic_agentos/protocol/mcp_manifest.py`** — deferred-load signed-manifest extractor. Resolves `cognic-pack-manifest.toml` via `Distribution.locate_file()` **WITHOUT importing the package code** per ADR-002 §"MCP STDIO threat model" gate 1. The deferred-load invariant is what keeps the manifest extractor safe against a malicious pack that side-effects on import; the cognic_test_mcp_pack fixture is intentionally import-poisoned (`raise AssertionError`) to make this invariant testable and **enforced as a load-bearing test**. Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/mcp_transports.py`** — two protocol-side transport classes:
  - **`StreamableHTTPTransport`** — Streamable HTTP transport with canonical MCP SDK `streamablehttp_client` wiring; `open_session` / `send` / `close_session` lifecycle backed by `AsyncExitStack`; transport `event_hook` contract emits the three `MCPTransportEventType` values (`session_open` / `session_close` / `send_error`) with **per-event hook-failure semantics** (intentionally non-uniform): `send_error` is safe-swallowed via `_emit_send_error_safe` so a broken audit hook can't mask `mcp_call_tool_timeout` / `mcp_transport_send_failed`; `session_open` is fail-closed (hook exceptions re-raise after the AsyncExitStack is closed); `session_close` is best-effort. **R2 P2 #1**: when `stack.aclose()` raises during a failed-open cleanup, the cleanup error is suppressed (logged token-free with `cleanup_error_type` only) so the original `mcp_session_open_*` exception surfaces unchanged. **R2 P2 #2**: send-error warning logs scrub hook exception text from log payloads.
  - **`StdioTransport`** — Sprint-5 Decision Lock realisation: a non-launching refusal stub. Three transport methods (`open_session` / `send` / `close_session`) all unconditionally raise `NotImplementedError` citing ADR-002 §"Sandbox dependency hard-block" + ADR-004 + the Sprint-8 hand-off. **No `register` method** (registry stays the single owner of `RegistrationOutcome` per Sprint-4 doctrine + R2 P2 #4); **no `_emit_*` methods** (audit emission for STDIO refusal lives on the registry side); **no `_event_hook` instance attribute**; **no `require_mcp()` call** (R3 P1 — class never references the SDK).
  - Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/mcp_host.py`** — admission-to-invocation orchestrator. ADR-014 transitional high-risk-tier gate: `MCPHost.call_tool` reads pack-manifest `risk_tier`; anything other than `read_only` / `internal_write` AND no `core/approval` engine (Sprint 5–13) → `MCPToolInvocationRefused(reason="tool_approval_engine_not_available")`; the high-risk pack still admits — only invocation is blocked. **`_DispatchContext`** (R2): split `last_acquired_token` (the token the host has obtained from `mcp_authz`) from `last_dispatched_(session, token)` (the token actually sent on a transport `send`); prevents a 401-retry from falsely combining the first dispatch's session_id with a never-sent token in the audit row. **`_emit_call_evidence`** (T11): single helper writes the parallel `audit.tool_invocation_*` audit row + `mcp_call` decision-history row from one canonical payload. **`_GENERIC_INVOCATION_ERROR_TAXONOMY = "mcp_orchestrator_error"`** closed-enum catch-all (R4) for the generic-Exception path so `type(exc).__name__` never appears in a closed-enum `error_taxonomy` field; the class name moves to a separate `error_type` payload field. **Per-tenant `list_tools` cache** with key `(tenant_id, server_id, manifest_scopes)` for cross-tenant isolation; bounded pagination with cap + cycle detection via opaque SHA-256 cursor fingerprints (`_fingerprint_cursor`); deep-copy of returned tool descriptors so callers can't mutate cache entries. **Token-free invariant** end-to-end: bearer `value` bytes never appear in audit / decision payloads or `__repr__`. Per-file coverage: 100/100.
