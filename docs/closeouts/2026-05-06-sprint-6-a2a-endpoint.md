## What ships in `feat/sprint-6-a2a-endpoint` after Sprint 6

### Seven new critical-controls modules

- **`src/cognic_agentos/protocol/a2a_authz.py`** — per-tenant pinned-token validator. Closed-enum 8-value `A2AAuthzReason` literal carries the audit-row taxonomy (`a2a_anonymous_refused`, `a2a_token_missing`, `a2a_token_malformed`, `a2a_tenant_mismatch`, `a2a_token_revoked`, `a2a_vault_read_failed`, `a2a_audience_mismatch`, `a2a_scope_insufficient`); Vault-read exception mapping per Sprint-5 T15 R1 P2 #2 doctrine (adapter-shape failures map to `a2a_vault_read_failed` rather than letting raw exceptions escape); `hmac.compare_digest` constant-time bearer-token comparison; `A2APinnedToken` is frozen+slotted with custom `__repr__` redaction so the bearer value never leaks via `__dict__` exposure or default-repr surfaces. Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/a2a_agent_cards.py`** — three-pass Agent Card validator + outbound-fetch verifier. **11-value** closed-enum `AgentCardValidationReason` literal (1 upstream-schema + 7 AgentOS profile + 3 JWS): T14 added the 7th profile gate `agent_card_profile_wave2_auth_required` for cards declaring `mtlsSecurityScheme` anywhere in `securitySchemes` — refused under Wave-1 bearer-token transport policy per A2A-CONFORMANCE.md §"Wave breakdown" (Wave-1 = per-tenant pinned bearer token; Wave-2 = mTLS; Wave-3 = verifiable credentials). Validation order is JWS-first (don't parse untrusted bytes), then forbidden-top-level-`url` pre-check, then SDK protobuf parse, then profile gates, with success emitting `audit.a2a_agent_card_validated` and refusal emitting both `audit.a2a_agent_card_rejected` + a `decision_history` row carrying the closed-enum reason. **Outbound fetch path** (`fetch_and_verify_outbound_card`) constructs the discovery URL via the constant well-known suffix concatenated to a strict-origin-only `target_origin` (rejects non-`http(s)` schemes / paths / queries / fragments / userinfo / non-string shapes BEFORE `httpx.AsyncClient.get` is awaited; `rejected_component` payload identifies the rejected URL component without echoing credential-like material) and pins `A2A-Version: 1.0` on every outbound `_http.get` per T14 Fix #2 (the spec's absent-header default is A2A 0.3 which AgentOS does not implement; without the header our outbound probes would land on the wrong wire contract on a 1.0-only target). Identity-routing critical: a forged or tampered card routes outbound traffic to attacker-controlled endpoints. Per-file coverage: 95.27/100.
- **`src/cognic_agentos/protocol/a2a_endpoint.py`** — inbound A2A receiver + task lifecycle state machine + cross-agent chain linkage. **6 ordered gates** in `handle()`: (1) version negotiation; (2) authentication; (3) Wave-2 feature refusal (runs BEFORE routing so a registered Wave-1 agent never receives Wave-2 traffic); (4) routing via plugin registry; (5) task creation + dispatch; (6) lifecycle transition + chain emission. Each gate refusal emits chained `a2a.task_refused` audit + `a2a_call` decision-history rows carrying parent-trace + locally-minted child-trace IDs so the cross-agent chain is walkable end-to-end including refusal legs. **Single-writer for `TaskState` transitions** via `_transition`; backwards / illegal moves raise `ValueError`. **Wave-2 classifier** (`_classify_wave2_feature`) is iterative + depth/node-bounded over the parsed JSON-RPC envelope, looking for any `parts` array entry with Wave-2 oneof fields (`raw` / `url`) or media-type prefixes (`image/` / `audio/` / `video/`); RecursionError from `json.loads` itself maps to the same fail-closed `payload_unscannable` sub-tag (T9 R5 P2 reviewer correction — extremely deep valid JSON would otherwise blow the C-side decoder's recursion budget BEFORE the iterative walker runs). The runtime SDK gate (`require_a2a()`) fires at construction time so mounting on a kernel image without `a2a-sdk` raises `A2ANotAvailableError` immediately rather than at first inbound traffic. Per-file coverage: 96.14/92.42.
- **`src/cognic_agentos/protocol/a2a_schema.py`** — pinned A2A 1.0 wire-format types via `a2a-sdk == 1.0.2`. PEP 562 lazy `__getattr__` re-exports the SDK's protobuf-generated message classes (NOT Pydantic — T6 R0 capture-time correction; the SDK ships protobuf via `MessageMeta`) under stable AgentOS names so the module imports cleanly without the SDK installed (admission-side per Sprint-5 R3 P1; first attribute access fires `require_a2a()`). `_REEXPORTED_TYPE_NAMES` frozenset of 9 entries keeps the surface explicit. Pinned digest constant + the upstream protobuf URL constant (the spec authors' canonical URL pinned to the v1.0.0 git tag, captured at T6 commit time alongside the SDK lock) drive the schema-drift CI gate (`tests/unit/protocol/test_a2a_schema_drift.py`, env-gated on `COGNIC_RUN_A2A_UPSTREAM=1`). Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/a2a_version.py`** — `A2A-Version` HTTP header parser + responder per ADR-003 §"Version negotiation". Closed-enum 6-case `A2AVersionOutcome` matrix: `accepted` (`1.0` matches pinned), `absent_rejected` (no header → spec interprets as `0.3` → rejected with `Supported-A2A-Versions: 1.0`), `legacy_rejected` (`0.x` → rejected), `higher_minor_degraded` (`1.<higher minor>` → processed with feature-degradation warning), `unsupported_rejected` (`2.x` or unknown → rejected), `malformed_rejected` (malformed header → spec parse error). `PINNED_VERSION = "1.0"` is the single source-of-truth used by both the inbound negotiator AND the outbound `a2a_agent_cards._http.get` callers per T14 Fix #2. Module is small + pure-functional but the doctrinal surface is wire-protocol-public — promoted to critical-controls at T15 R0 P2 #4 + R2 P2 #4. Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/a2a_errors.py`** — owns the spec wire `A2AErrorCode` literal (14 spec-defined codes — 5 JSON-RPC envelope errors + 9 A2A-specific) and the AgentOS `A2APolicyRefusalReason` literal (11 policy reasons including `unknown_target` / `wave2_feature_refused` / `agent_card_signer_not_allowlisted`) and the `_POLICY_REASON_TO_SPEC_CODE` mapping that drives the error-response builder. The split keeps the wire contract spec-conformant while still surfacing the rich AgentOS refusal vocabulary the audit + decision-history chains depend on (mirrors the Sprint-5 MCP authz pattern). Drift in any of these changes what remote A2A callers see; promoted to critical-controls at T15 R3 P2 #2 — earlier draft kept this non-critical because the module is "just enums"; reviewer correctly flagged that the mapping IS wire-protocol contract. Per-file coverage: 100/100.
- **`src/cognic_agentos/protocol/ui_events.py`** — typed UI event taxonomy + emit-hook layer per ADR-020. **All 11 Wave-1 Pydantic event-family models seeded in Sprint 6** (R0 P2 reviewer correction — schema covers full ADR-020 §"Event taxonomy (Wave 1)" regardless of which sprint wires the emit hooks): `agent_run.{started,progress,completed,failed,cancelled,paused,resumed}`, `tool_call.{requested,approved,denied,started,progress,completed,failed}`, `subagent.{spawned,completed,failed,recursion_capped}`, `approval.{pending,granted,granted_second,denied,expired}`, `artifact.{started,chunk,completed}`, `interrupt.{requested_by_agent,requested_by_operator,acknowledged}`, `frontend_action.{submitted,accepted,rejected}`, `memory.{recall_started,recall_completed,forget,redact}`, `decision_audit.event_appended`, `policy.{decision_evaluated,bundle_loaded}`, `kill_switch.{flipped,reverted}`. The discriminated `UIEvent` union is **two-level** (R1 P2 #1): per-family inner unions discriminated on `type`, top-level union discriminated on `family` (a flat single-discriminator union fails Pydantic adapter creation because each family has multiple event classes). Emit-hook protocol (`UIEventHook`) wires **3 families in Sprint 6** — `tool_call.*` (mirrors Sprint-5's `audit.tool_invocation` series), `artifact.*` (mirrors Sprint-6 T11's `a2a.artifact_prepared`), `decision_audit.event_appended` (the universal generic mirror emitted from BOTH `AuditStore.append` AND `DecisionHistoryStore.append`, with `data.chain_id` discriminating `"audit_event"` from `"decision_history"` so audit-only sources still mirror to UI). All three hook dispatch loops snapshot the registry via `tuple(self._hooks)` at entry per R2 P3 so a self-registering hook can never extend the iteration target indefinitely. Per-hook deep-copy isolation (R1 P2 #2 + #3) — `frozen=True` is shallow; explicit deep-copy prevents one hook from mutating data another hook sees. The other 8 families have model-only stubs; their emit hooks land in their owning sprints per the ADR-020 phase table. **No SSE endpoint** in Sprint 6 (ADR-020 schedules SSE for Sprint 7B). Per-file coverage: 100/100.

### Four small endpoint modules (admission-side, not on the critical-controls list)

- **`src/cognic_agentos/protocol/a2a_streaming.py`** — A2A 1.0 streaming wire-format adapter. Builds protobuf `StreamResponse` envelopes (oneof `task` / `message` / `status_update` / `artifact_update`) via the T6 schema-module re-export boundary; lifecycle transitions ride `TaskStatusUpdateEvent` whose `TaskStatus.state` is one of the 9 SDK `TaskState` enum values; artifact streaming rides `TaskArtifactUpdateEvent` with `append` / `last_chunk` flags. Wire encoding via `google.protobuf.json_format.MessageToJson` (no hand-rolled JSON). Public boundary is the AgentOS-side `StreamState` Literal (8 values mirroring SDK `TASK_STATE_*` 1:1). NOT portal/UI SSE — that's Sprint-7B per ADR-020.
- **`src/cognic_agentos/protocol/a2a_artifacts.py`** — artifact reference generator. Large outputs (PDFs, evidence packs, JSON > 64 KiB) stored via Sprint-4's `LocalObjectStoreAdapter` and returned as `ArtifactRef(uri, sha256, size_bytes, mime_type)`; small payloads remain inline; per-tenant retention configurable via `Settings.a2a_artifact_retention_seconds`. Emits `a2a.artifact_prepared` audit row that the UI events `artifact.*` hook mirrors per ADR-020.
- **`src/cognic_agentos/protocol/a2a_capability_negotiation.py`** — `GET /api/v1/a2a/capabilities` endpoint backing module. Reads the canonical FLAT `[tool.cognic.a2a]` block from pack manifests; returns a Wave-1-filtered `A2ACapabilities` response with Wave-2 declarations forced to false + surfaced in `deferred_wave2_features`. Strict bool typing — truthy non-bool shapes (e.g. `"false"`) treated as `False`. Subset of manifest declarations; never broader.
- **`src/cognic_agentos/protocol/a2a_cancellation.py`** — task cancellation primitive. Standalone `A2ATaskCancellationHandler(endpoint)` with the T9 endpoint injected; **never mutates `TaskState` directly** (single-writer invariant pinned by an AST-walk regression). `cancel_task(*, task_id, request_id, tenant_id, parent_trace_id=None)` delegates the lifecycle flip to `endpoint._transition_async(..., new_state=TaskState.CANCELLED)`. Refusal paths chain-linked per T11 R1 P2 #3: unknown task → spec `task_not_found`; terminal-state task → spec `task_not_cancelable`; both emit `a2a.task_refused` audit + `a2a_call` decision-history rows.

### Two architecture-test backstops (T4)

- **`tests/architecture/test_a2a_no_subprocess.py`** — static-AST scan of every module under `protocol/a2a_*.py` (recursive). Refuses any `subprocess` / `os.exec*` / `os.spawn*` / `os.posix_spawn*` / `os.system` / `os.popen` / `asyncio.create_subprocess_exec` / `asyncio.create_subprocess_shell` / `multiprocessing.Process` / `shell=True` import or call. **Sentinel tightened in T16 closeout (this commit) from `>= 0` to `>= 9`** — Sprint 6 ships 10 a2a_*.py modules; the 9-floor leaves room for one rename without tripping but trips before the parametrized scan starts vacuously passing on accidental module deletion.
- **`tests/architecture/test_a2a_no_caller_controlled_url.py`** — static-AST scan asserting that NO module under `protocol/a2a_*` calls `httpx.AsyncClient.{get,post,put,patch,delete,request,send}(url=<x>)` where `<x>` traces to a function parameter, an inbound-request attribute chain, a function-parameter-rooted attribute chain, or a model-output string. Allowed URL sources are exactly four: string literal, `Settings.a2a_*_url` attribute access, hardcoded well-known suffix concatenated to a non-caller-controlled origin, or attribute access rooted at a verifier-output name (`verified_card` / `verified_agent_card`) with chain containing `supported_interfaces` or ending with `.url`. URL extraction is method-aware (`client.request(method, url, ...)` reads URL from positional arg 1; `client.send(request_obj)` reads URL from the Request object — statically traced as "unknown" since runtime canary takes over). Three self-tests pin the collector + the URL-source classifier.

- **Token-free invariant.** `A2APinnedToken.value` (the raw bearer bytes) is never logged, never serialised in audit / decision payloads, never visible in `__repr__`. Frozen+slotted dataclass disables `__dict__` access; custom `__repr__` redacts `value`. - **Production-grade rule.** No mocks in runtime paths. `A2AAgentCardVerifier` uses real `httpx.AsyncClient` for outbound card-fetch HTTP I/O at runtime; `A2AAuthzClient` uses real Vault SecretAdapter at runtime. The fixture pack's mocked-trust-gate / mocked-secret-adapter approach lives **only at the unit test boundary**; production code paths remain real and swappable per AGENTS.md §"Production-grade implementation rule".
- **Plugin discipline (ADR-001).** No agents, tools, skills, UI, or bank overlays added. All work sits under platform-primitive (`protocol/a2a_*` + `protocol/ui_events`), policy-evidence (none added — A2A-side has no Rego seed in Sprint 6; the existing Sprint-4 + Sprint-5 Rego bundles cover supply chain + sampling), and test-fixture (`tests/fixtures/cognic_test_agent_pack/`, `tests/fixtures/a2a-conformance/`) layers.
