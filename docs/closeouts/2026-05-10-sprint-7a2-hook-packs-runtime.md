## What ships in `feat/sprint-7a2-hook-packs-runtime` after Sprint 7A2

### SDK surface (Sprint-7A2 T2)

- **`src/cognic_agentos/sdk/hook.py`** — public Hook authoring API:
  - `Hook` ABC. Subclasses declare `hook_id: ClassVar[str]` + a single `phase: ClassVar[HookPhase]` (one Hook subclass binds to exactly one phase) and override `async def _invoke(self, context: HookContext, payload: bytes) -> HookResult`. The SDK's public `async def invoke(self, context: HookContext, payload: bytes) -> HookResult` is `@typing.final` (mypy) AND `__init_subclass__`-guarded (mirrors `Tool.invoke()` from Sprint-7A T2 doctrine — any ancestor in MRO defining `invoke` directly raises `TypeError` at class-creation time so mixin-smuggling cannot bypass the SDK's context / payload / result validation seam). `invoke()` validates context + payload shape BEFORE delegating to `_invoke`; validates the returned `HookResult` shape + decision-↔-fields invariants AFTER.
  - `HookContext` — `hook_id` + `phase` + `pack_id` (the CALLING pack, not the hook pack) + `tenant_id` + `request_id` + `trace_id` / `parent_trace_id` (cross-agent chain linkage) + `manifest_data_classes: tuple[str, ...]` + `manifest_purpose: str` (calling pack's data-governance snapshot at admission). Frozen + slotted dataclass. Deliberately does NOT carry the payload bytes — payload is passed as a separate `_invoke()` argument so context is safely loggable; pinned by the AST-walk regression `tests/architecture/test_hook_payload_never_logged.py`.
  - `HookResult` — `decision: HookDecision` (closed-enum `Literal["pass", "redact", "mask", "refuse"]`) + `redacted_payload: bytes | None` (MUST be bytes for `redact`/`mask`; MUST be None for `pass`/`refuse`) + `policy_reason: str | None` (MUST be a non-empty string for `refuse`; MUST be None for `pass`/`redact`/`mask`) + `audit_metadata: dict[str, Any]` (token-free metadata the hook attaches to its audit row). Frozen + slotted; the decision-↔-fields invariant is enforced by `Hook.invoke()` AFTER `_invoke` returns (violations raise `HookResultShapeError`).
  - `HookPhase` — closed-enum `Literal["dlp_pre", "dlp_post"]` Wave-1 (sourced from `cognic_agentos.cli._governance_vocab`); future phases (memory governance / escalation / egress) extend the enum in their owning sprints.
  - Error hierarchy: `HookError` (base — single class the dispatcher catches) → `HookContractError` (SDK contract violations; routed to `hook_malformed_result`; ALWAYS fail-closed) → `HookContextError` / `HookPayloadError` / `HookResultShapeError`. Cannot be smuggled past the malformed-result gate via the `fail_open_exception` carve-out (T7 R1 P2 fix established symmetric exception ordering across the loader/constructor + post-instantiation try blocks; pinned by AST self-tests + threat-model-revert verification per `feedback_security_regression_hardening.md`).

### Runtime hook engine (Sprint-7A2 T6 + T7 + T8)

- **`src/cognic_agentos/packs/hooks/registry.py`** (T6, CRITICAL CONTROLS) — verified-hook admission gate. `HookDeclaration` frozen dataclass enforces the Wave-1 fail-policy invariant at construction time (`fail_policy="fail_open"` requires non-empty `fail_open_exception`). Registry public surface (3 methods): `register_pack(verified_pack)` keys hook entries by `(hook_id, phase)` + pack identity + signed-artefact digest from the trust gate's verified bundle and refuses fail-closed on duplicate-ID across packs / stale digest / cross-pack-conflict scenarios; `snapshot()` returns an immutable `MappingProxyType` view over a dict-snapshot of the current registry state; `get_phase_hooks(phase)` returns the ordered tuple of `HookEntry` snapshots filtered to that phase. The dispatcher (T7/T8) consumes registry snapshots via `get_phase_hooks` (taken once per dispatch call so subsequent `register_pack` calls do not retroactively appear) — admission and dispatch are decoupled so the tuple-snapshot semantics survive registry mutation during dispatch (mirrors Sprint-6 `protocol/ui_events.py` doctrine).
- **`src/cognic_agentos/packs/hooks/dispatcher.py`** (T7, CRITICAL CONTROLS) — runtime decision engine. Public surface: `HookDispatcher.dispatch(*, phase, payload, context_template)` (phase-wide; iterates every hook the registry has for the phase) + `HookDispatcher.dispatch_for_pack(*, phase, declared_hook_ids, payload, context_template)` (per-pack; the calling pack's `[data_governance].dlp_*_hooks` list is passed as `declared_hook_ids`, and only hooks whose ID is in that sequence run for this pack). Both methods consume `HookRegistry.get_phase_hooks(phase)` once at dispatch entry to obtain a tuple snapshot, then iterate against that snapshot. When a hook_id in `declared_hook_ids` is not registered for the requested phase, `dispatch_for_pack` raises `HookDispatchSelectionError` — a structured `RuntimeError` (NOT a closed-enum) carrying `hook_id` + `phase` attributes; the closed-enum terminus is `dlp_hook_id_unresolved`, surfaced one layer up after `DLPGuard` catches the exception and translates it. The exception fires only AFTER the dispatcher's payload-budget check passes, so an oversized + unresolved-id input correctly routes to `hook_payload_unscannable` rather than to the selection error (T8 R1 P2-2 doctrine: budget-check-before-lookup precedence preserved by delegating from DLPGuard rather than pre-validating). Closed-enum 5-value `HookFailureMode` literal: `hook_timeout` / `hook_exception` / `hook_malformed_result` / `hook_policy_refused` / `hook_payload_unscannable`. Failure-mode mapping:
  - `asyncio.TimeoutError` → `hook_timeout` (always fail-closed; timeout fires OUTSIDE the hook's catch boundary so fail-open never applies).
  - `HookContractError` (any subclass) → `hook_malformed_result` (always fail-closed; SDK contract violations are programming errors). Pre-instantiation `except HookContractError` matches BEFORE generic `except Exception` so a malicious declaration cannot smuggle a contract violation past the malformed-result gate by naming `HookContractError` (or any subclass name) as `fail_open_exception` — pinned by AST self-tests + threat-model-revert verification (T7 R1 P2 fix; symmetric exception ordering across loader/constructor + post-instantiation try blocks).
  - Any other `Exception` → `hook_exception`, with fail-open carve-out: if `entry.fail_policy="fail_open"` AND the exception's class name (walked through MRO per `dispatcher.py:567-569`) matches `entry.fail_open_exception`, treat as `decision="pass"`. **Wave-1 reality:** the validator refuses every `fail_open` declaration at the build-time boundary, so the carve-out is wired-but-unreachable through the manifest pipeline until the matching build-time shape lands in a follow-up sprint.
  - Hook returned `decision="refuse"` → `hook_policy_refused`.
  - Payload exceeded `Settings.hook_max_payload_bytes` budget → `hook_payload_unscannable` (size-check is a dispatcher precondition; never invokes a hook for an over-budget payload).
  - **Audit invariant:** every finding records the SHA-256 `policy_input_digest` of the **ORIGINAL** governed payload (transformations never enter the audit chain); hook payload contents themselves are never logged. Pinned by `tests/architecture/test_hook_payload_never_logged.py` — an AST sweep over every Python file under `src/cognic_agentos/packs/hooks/` (rglob discovery, so newly-added hook modules are picked up automatically) that bans seven payload-leak shapes flat across each module: bare `print(...)` calls, `logging.<info|debug|warning|error|critical|...>(...)` module-level calls, `<logger>.<info|debug|warning|error|critical|exception|...>(...)` attribute-call shapes, f-string formatted-values referencing the `payload` Name, `str(payload)` / `repr(payload)` / `format(payload)` stringification, `payload.decode(...)`, and `"%s" % payload` / `"...".format(payload)` formatting. Allowed: `hashlib.sha256(payload)` digest computation (the loadbearing reason payload enters the dispatcher), `len(payload)` size check, delegation as positional / keyword arg to `Hook.invoke` / `hook.invoke` / `instance.invoke`, assignment / re-assignment, and length / boolean comparison.
- **`src/cognic_agentos/packs/hooks/dlp_integration.py`** (T8, CRITICAL CONTROLS) — DLPGuard adapter wrapping `HookDispatcher.dispatch_for_pack` with `dlp_pre` / `dlp_post` phase semantics. Closed-enum 3-value `DLPRefusalReason`: `dlp_hook_id_unresolved` / `dlp_dispatcher_failed` / `dlp_dispatcher_refused`. The `dlp_hook_id_unresolved` surface is exclusively a runtime closed-enum — translated from the dispatcher's `HookDispatchSelectionError` whenever a calling pack's manifest-declared hook reference does not resolve against the registry snapshot. ADR-017 line 97 enforcement boundary. Two T8 doctrine memories born from R1 P2 fixes:
  - **Refusal-payload contract divergence** (`feedback_refusal_payload_contract_divergence.md`): wrapper's "what payload to return on refusal" contract diverges from engine's last-seen-payload contract. DLPGuard returns the ORIGINAL governed payload on every refusal, never `dispatch_result.final_payload` (which would be a partial-redaction leak). Pinned by transformation-then-refusal regression.
  - **Delegate-first preserves precedence** (`feedback_delegate_first_preserves_precedence.md`): wrappers MUST delegate first + catch typed exceptions + translate, NEVER pre-validate. DLPGuard's pre-validation pass would have run lookup BEFORE the dispatcher's payload-budget pre-check, so oversized + unknown-id input would surface as `dlp_hook_id_unresolved` instead of the engine's correct first-precedence `hook_payload_unscannable` / `dlp_dispatcher_failed`. Pinned by an oversized + unknown-id regression that asserts `hook_payload_unscannable`.
