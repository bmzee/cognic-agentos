## New doctrines established Sprint-8A

- **`feedback_canonical_artifact_not_oss_substitute`** — when a spec/ADR designates a canonical image/binary as security enforcement point (cognic/sandbox-egress-proxy, cognic/sandbox-runtime-*, signed pack bundles), it IS a real shippable artifact, NOT placeholder, NEVER silently substituted by OSS equivalent. OSS (tinyproxy/mitmproxy/envoy) is fine INSIDE the canonical image (with cosign+SBOM) OR as clearly-named local fixture behind env flag (`COGNIC_USE_LOCAL_FIXTURE_PROXY=1` + refused in production profile). Missing canonical artifact → env-gated test skips/fail-loud with structured message naming missing ref, NEVER silent swap. - **`feedback_sandbox_network_isolation_precision`** — egress proxy MUST live INSIDE the sandbox netns OR use Unix socket OR custom bridge with proxy-only egress; phrasing it as "sidecar bound to sandbox loopback" is impossible under `--network none`. Wave-1 AgentOS doctrine: HTTP/HTTPS-only through AgentOS-controlled proxy endpoint; sandbox has no direct external network route; non-HTTP protocols refused in Wave 1.
- **`feedback_precise_security_terminology`** — name the actual mechanism, not its colloquial overload. Example: ADR-004's "DinD" reads as nested-privileged-daemon (acceptable trust posture) but the implementation is Docker-sibling-on-host-socket (compromised-AgentOS = root-on-host). - **`feedback_immutable_runtime_images_no_dynamic_install`** — production sandboxes / runtimes use digest-pinned + cosign-signed + SBOM-scanned images only; dynamic apt-get/pip at create-time rejected for production (dev profile only); tools needing specialised binaries either declare a canonical AgentOS-catalog image OR ship their own signed per-pack image; banks reject runtime-install patterns at first security audit because evidence-pack export per ADR-006 can't capture the binary set.

- **`feedback_git_diff_check_untracked`** — plain `git diff --check` is VACUOUS on untracked files (silently reports clean even with trailing whitespace); for new specs/plans/ADRs before first commit use `git diff --no-index --check /dev/null <path>` OR `git add -N <path>` then plain `git diff --check`.
