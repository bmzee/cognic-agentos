## Design locks (Q1-Q5 + the extra session_id-authorization lock)

The brainstorming session of 2026-05-18 locked 5 design calls plus 1 extra lock; all carried through to implementation:

- **Q1 — workspace-tar snapshot.** The checkpoint mechanism is a workspace-tar (not an overlay-fs diff), which gives `DockerSibling` + `KubernetesPod` identical backend parity. Propagates through §3 / §4 / §7 / §10 of the spec.
- **Q2 — dedicated `checkpoint_purged` audit event.** Purge is its own first-class `sandbox.lifecycle.checkpoint_purged` event with a 4-value closed-enum `purge_reason`, not a sub-case of `destroyed`.
- **Q3 — `admit_policy` revalidation at wake time.** `wake()` re-runs the Stage-2 admission pipeline; a policy that no longer admits surfaces `sandbox_wake_policy_revalidation_failed`. Propagates through §3.1 / §3.3 / §5 / §8 / the T8 chain-verifier extension.
- **Q4 — fail-loud Vault; NO `CredentialAdapter` extension in 8.5.** The existing Sprint-8A `CredentialAdapter` surface is exactly `async def fetch_secret(path) -> str | None`; the Sprint-8A negative pins at `tests/unit/sandbox/test_credential_adapter_stub.py:24+99-116` forbid `mint_lease` / `revoke_lease` until Sprint 10. Sprint 8.5 honors those pins — `sandbox/credentials.py` is **UNCHANGED** (verified empty diff vs `main`). `CheckpointMetadata` still carries `vault_lease_refs: tuple[VaultLeaseRef, ...] = ()` for forward-compat wire shape (always empty in 8.5 — no vault-bearing session can be created today, so none reaches checkpoint). **T5 was DELETED** from the task arc as a result — the planned lease-re-issue task had no contract to implement.
- **Q5 — `wake()` carries caller identity.** The backend Protocol signature is `wake(session_id, *, actor, tenant_id) -> SandboxSession` so Sprint 10.5's scheduler can wrap it with the same admission identity seams as `create()`. ADR-004 §84's `sandbox.wake(session_id)` stays as module-level shorthand.
- **Extra lock (§2.6) — `session_id` alone is NEVER authorization.** `wake()` loads `CheckpointMetadata` keyed by `<tenant_id>/<session_id>/...`; a cross-tenant wake attempt cannot find the checkpoint at the caller's prefix, and a tenant-id mismatch fails closed with `sandbox_wake_tenant_mismatch`. This pre-empts the entire "guess a session_id from another tenant + call wake()" attack class.
