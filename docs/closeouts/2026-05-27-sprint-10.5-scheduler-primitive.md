## Doctrine outcomes locked in 10.5

### 1. Option A ‚Äî SchedulerPolicy owns Rego ONLY (T9 LOCK)

`SchedulerPolicy` is the single Python boundary that consults the `scheduler.rego` bundle at `data.cognic.scheduler.admit.allow`. It does NOT consult `core/emergency/*` (quota or kill_switch). Those operational gates live in `SchedulerEngine.submit()` ordering ‚Äî invoked via the consumer-owned `QuotaInterrogator` + `KillSwitchInterrogator` Protocols. The doctrine conflation rejected at T9: kill_switch + quota are **operational real-time emergency surfaces (ADR-018)**, not **policy-as-code declarative bundles (ADR-015)**.

### 2. T11 SandboxAdapter ‚Äî atomic create+destroy pair

Round-2 P1 rejected the round-1 two-callable signature (`sandbox_create_fn` + `sandbox_destroy_fn` as separate kwargs) because production code could pass `create_fn` WITHOUT `destroy_fn` ‚Äî leaks unrepresentable required by the type system. Replaced with a single `SandboxAdapter` Protocol object carrying both methods. Now the "create succeeded but destroy was never provided" bug class is **unrepresentable at the type level**.

### 3. Substrate independence ‚Äî scheduler never imports from sandbox or emergency

Two AST architectural-arrow guards (`test_architecture_no_emergency_import.py` + `test_architecture_no_sandbox_import.py`) pin the boundaries. PEP-328 relative-import resolver with 5 self-tests + off-by-one bug surfaced + closed in the SAME commit. Real conformers bind at AgentOS app startup via the DI binder, NOT at module-import time.

### 4. Doctrine candidates surfaced + closed in 10.5

- **6 reviewer rounds on doc-split-state consistency at Z1b** ‚Äî chased every "X promised; tree shows Y" discrepancy across BUILD_PLAN + AGENTS.md + plan-of-record + 10.5/10.6 spec headers + ADR-022's "Cross-ADR amendments" paragraph + closeout-report scoping. Refines `[[feedback_plan_reconciliation_in_tree_before_halt]]` to multi-doc-state coordination at sprint-close.
- **Subprocess env lockstep via test-only drift detector** (T8 round-1 P2) ‚Äî when two production modules must share a subprocess-env constant, neither imports from the other at runtime; instead a test imports from both and asserts equality. Reused the pattern from `[[feedback_drift_detector_test_only_no_runtime_import]]`.
- **Consumer-owned Protocol seams + fail-loud sentinels** (T5-T11) ‚Äî when a planned CC consumer depends on a yet-unlanded downstream module, declare the dep Protocol in the consumer module + ship a fail-loud `_Null*` sentinel; downstream sprint binds a structurally-conforming real implementation at the DI seam. Confirmed pattern from `[[feedback_consumer_owned_protocol_for_unlanded_dep]]`.
