## Scope

Sprint 11.5a ships `core/memory/` + `core/dlp/` — the **governed memory
substrate** every Layer C agent uses (ADR-019), so banks get the same
compliance guarantees on what an agent *remembers* that they already have on
tool calls and LLM invocations. Three tiers (`scratch` / `task` / `long_term`),
**default-deny `long_term`**, per-write data-class + purpose + consent
enforcement, chain-linked audit, a regulator-erasure-ready schema, and the SOTA
**memory-block** shape (Letta/Hermes-style labeled core memory) as a first-class
governed `long_term` projection.

**Posture (honesty marker, held throughout):** the `MemoryAPI` is **DI-tested,
not harness-injected** — memory writes are not production-wired until the
erasure/redaction/export lifecycle lands in **11.5b**. Same primitive-done /
live-dispatch-deferred posture the user ratified for the 11b sub-agent primitive.

## Decisions locked

- **DI-tested, not harness-injected** — the substrate is closed; live dispatch wiring is intentionally 11.5b.
- **Value never enters the chain (write vs read payloads differ).** `memory.write` is appended via `append_with_precondition` (atomic with the `memory_records` row); its payload carries tier / data_classes / purpose / retention / record_id / subject_ref / block_kind + a **redacted-value-digest** only, stamped ISO `A.7.4` / `A.8.2` / `A.8.5` / `A.10.2`. `memory.read` is a plain `append` (a read produces no value) carrying **read-shaped** metadata only — keyed reads `{tier, purpose, subject_ref, hit, record_id}`; the `list_for_subject` / `recall_episodes` enumerate `{op, …, subject_ref, hit, count}` — and **never a value or value-digest**, stamped ISO `A.7.4` / `A.8.2`. The JSONB value lives in `memory_records`; NO `core/canonical.py` / `core/decision_history.py` schema edit (payload-only linkage).
- **Tenant- AND agent-scoped reads** — `get` / `list_for_subject` / `list_blocks` take required `tenant_id` + `agent_id`; a record belongs to the writing agent (the block singleton identity makes it explicit). Cross-agent reads within a tenant are unrepresentable — TM-revert-proven in review: without the `agent_id` filter, agent A's `read_block` returned agent B's block.
- **Strict sub-agent rule** — `is_subagent` refuses ALL durable (`task` / `long_term`) `remember` / `recall` / `upsert_block` / `read_block` / `recall_episodes`; children are `scratch`-only; durable is parent-mediated (snapshot seam → Wave-1.5).
- **Default-deny + fail-loud** — long_term / cross_subject / restricted-class default-deny via Rego (OPA error → deny at that decision point only); the kill-switch `_Null` sentinel raises `NotImplementedError` (no permissive hidden default); `scratch` is fail-closed on Redis-unavailable (no un-erasable Postgres fallback — Cut-A).
- **Recall threads the STORED write purpose** (review P1) — `recall` / `read_block` run pre-read authz, fetch, THEN the purpose matrix vs `hit.purpose`; passing `None` against the default-deny bundle would refuse every recall.
- **Vector is a standalone DI-tested primitive** (review F1) — NOT wired into `recall_episodes`; `recall_episodes` fails loud on `similarity_threshold > 0.0` (vector ranking → 11.5b). Episode↔`decision_history` linkage reads the table via the store engine (no new public read API — review F2).
