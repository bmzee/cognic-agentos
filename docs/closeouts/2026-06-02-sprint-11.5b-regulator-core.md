## Scope

Sprint 11.5b makes the 11.5a memory substrate **regulator-operable**: it ships the erasure lifecycle (`forget` / `redact` / regulator-erasure), the tombstone reaper, the **real** Redis `memory.write_freeze` kill-switch (11.5a shipped only the fail-loud seam), and the scratch→Postgres durability fallback that 11.5a deliberately deferred under Cut-A. This closes the **regulator-operability** gap in the DI-tested substrate: a bank can now erase, redact, and emergency-freeze governed memory through the same hash-chained, value-never-in-chain audit path the substrate already gave reads and writes (the `MemoryAPI` itself stays DI-tested, not harness-injected — see the posture marker below).

**Posture (honesty marker, still held):** the `MemoryAPI` remains **DI-tested, not harness-injected** — production wiring of the API into the agent harness stays deferred past 11.5 (same primitive-done / live-dispatch-deferred posture as the 11b sub-agent primitive and 11.5a). 11.5b delivers the OS-side *primitives* (kill-switch, adapters, reaper, routing) and the app-lifespan opt-in for the reaper; the harness injection of `MemoryAPI` itself is not in scope.

## Decisions locked

- **11.5b/11.5c valve split (lock 1).** The original single 11.5b was split at the 2026-06-01 valve into **11.5b regulator-core** (this sprint) + **11.5c surfaces**; authoritative in ADR-019 §"Implementation phases" + the plan footer.
- **Reaper is a standalone module + app-lifespan opt-in (lock 2).** `core/memory/reaper.py` (not folded into storage/api), started by the `create_app` lifespan — the same `CheckpointReaper` precedent (Sprint 8.5); off-gate per Doctrine F since the substantive retention-floor enforcement is the on-gate `adapter.purge_expired`.
- **Export deferred to 11.5c reusing ADR-016 Sigstore (lock 3).** Not started in 11.5b; pre-locked to reuse the `protocol/supply_chain.py` Sigstore-bundle discipline.
- **Value never enters the chain — extended to the erasure trio.** `memory.forget` (tombstone) and `memory.regulator_erasure` (purge) chain rows carry metadata only (record_id / reason / subject_ref / custody refs); `memory.redact` carries the **new-value digest only**, never the new value. The raw value lives in `memory_records`; no `core/canonical.py` / `core/decision_history.py` schema edit (payload-only linkage), exactly as 11.5a established for `memory.write` / `memory.read`.
- **Real kill-switch fails closed on every uncertain input.** A non-bool `frozen`, any missing custody field, or a malformed doc → treated as frozen; a malformed read **poisons the cache** `(True, now)` so a later outage inside the cache TTL still refuses (a stale `False` would have served writes during a compliance freeze).
- **Scratch fallback is unavailable-only + read-after-recovery.** The `RoutingMemoryAdapter` falls back to Postgres ONLY on `MemoryBackendUnavailable.unreachable=True` (a Redis write/read *bug* propagates, no fallback); scratch reads are Redis-first/hit-wins, then PG on miss-or-unavailable, so a fallback row written during a past outage stays readable after Redis recovers.
- **Regulator-erasure custody is row-bound.** The command's `subject_id` MUST match the purged row's `human:<id>` subject; a mismatch refuses and deletes nothing.

## Review lessons (R-rounds — green-but-wrong prevented)

The arc ran reviewer-heavy; each lesson is pinned by ≥1 load-bearing (TM-revert-proven) regression:

- **Stateful cache-poisoning** (T3) — a malformed kill-switch read that returned `True` but left a stale `False` cache would serve writes during a later in-TTL outage; fixed to cache `(True, now)`.
- **Tenant/agent isolation** (T4) — the erasure mutators scope SELECT **and** UPDATE/DELETE by `tenant_id` + `agent_id`; a `record_id` is not an authz boundary, so a wrong tenant/agent erasure deletes nothing.
- **Migration-only constraints** (T4) — `uq_memory_block_singleton` is a partial unique index that lives in the Alembic migration, NOT `create_all()`; redact must tombstone-then-insert, and the regression runs against the migrated DB (not `create_all`).
- **Scratch key collision** (T8) — a `:`-joined scratch key aliases across subjects when an ID/key contains `:`; replaced with `sha256(canonical_bytes([tenant, agent, subject, key]))`.
- **Redis envelope validation** (T8) — the scratch read parser shape-validates every field (str-raw normalization, non-dict, record_id/created_at/data_classes/purpose/block_kind) and fails closed `unreachable=False`, never a raw exception nor an invalid `MemoryHit`.
- **Verify-floor-at-promotion-time** (Z1b) — the gate ran against fresh full-suite coverage IN the promoting commit, caught the `_routing.py` miss **and** the sibling `storage.py` regression, and repaired both in the same commit.
