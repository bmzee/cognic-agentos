## Scope

Sprint 11.5c ships the **surfaces** around the 11.5a/11.5b governed-memory substrate: the `export` op + the portal `/memory` API + the UI `memory.*` emit-hooks + the `learning_surface` manifest validator + the vector→real episodic-recall wiring. The substrate became *operable* in 11.5a/b (reads, writes, erasure, kill-switch); 11.5c makes it *reachable* — a bank examiner can export a subject's records to a retention-disciplined archive, an operator can drive forget/redact/export through an HTTP surface gated by RBAC + Human-only-decisions, the UI can render the `memory.*` audit stream, and the episodic recall view gains real vector ranking — all through the same hash-chained, value-never-in-chain audit path.

**Posture (honesty marker, still held):** the `MemoryAPI` remains **DI-tested, not harness-injected** — production wiring of the API into the agent harness stays deferred past 11.5 (same primitive-done / live-dispatch-deferred posture as 11.5a, 11.5b, and the 11b sub-agent primitive). 11.5c delivers the OS-side *surfaces* + the `create_app(memory_api_factory=…)` opt-in; the harness injection of `MemoryAPI` itself is not in scope.

## Decisions locked

- **One arc, valve PASSED (D3).** T7 (vector→real recall — the D3 valve candidate) stayed within the bounded shape (query + vector recall + governed-set authz-intersection + score filter + non-restricted long_term best-effort index). NONE of the split triggers fired — `search` still calls `client.search(..., filter=None)` (no server-side qdrant `Filter`), no embedding-residency controls, no new storage read (only the existing `list_for_subject`), fakes-only tests. **No 11.5d split.**
- **Export = retention-discipline only (F3).** Serialize the authorized `(agent, subject)` records to canonical bytes (values included — caller holds `memory.export.read`), persist with a caller-supplied bucket + retention (7-year ADR-016 floor, **inline-mirrored in `core/config.py`** with a test-only drift detector — NOT a `core → protocol` import). **Cosign-signing the archive is DEFERRED.** Put-first fail-closed; the distributed-atomicity boundary (append-fails-after-put → conservative orphan, no compensating delete under WORM retention) is documented + pinned.
- **Portal = surface-only, agent-scoped (F2).** Per-request `MemoryAPI` factory (never imports `core.memory.storage`); RBAC via `RequireScope`; `RequireHumanActor` on the destructive `regulator_erasure` + `export` paths; required non-empty `(agent, subject)` selectors (422 before any core call). **Whole-subject cross-agent erasure DEFERRED.**
- **Recall = client-side filter (F1 + D2).** `recall_episodes` gains `query`; `threshold > 0.0` intersects the vector hits with the governed agent-scoped active long_term set (an ungoverned hit is dropped — the authz is *structural*: Episodes are built from the governed set). Index-on-write only for non-restricted long_term records, best-effort. **Server-side qdrant `Filter` translation DEFERRED.**
- **Value never enters the chain — extended to the surfaces.** `memory.export` carries archive sha256 + key + count + retention only (the values live in the object-store archive); the `memory.*` UI projectors pass the already-value-free chain payload; the export-persistence-failure message carries sha256 + len + key + exc-class, never raw bytes.
- **`memory_approval_engine_not_available` stays (T8 hand-off).** The transitional Step-7 refusal is the sole memory write-approval surface until 13.5 wires `core/approval/engine.py`; the refusal reason is wire-public (count- and exact-set-pinned at 17), so removing it would be a coordinated ADR + wire-vocab amendment, out of scope.

## Review lessons (R-rounds — green-but-wrong prevented; each pinned by a TM-revert-proven regression)

- **Scratch-in-enumerate bug class** (T3 + T4 + T4.1) — `export` / `list_records` / `list_for_subject` all passed scratch fallback rows (post-11.5b `put_scratch_fallback`) the enumerate gate never authorized; an ephemeral scratch value would have been 7-year-retained (export) or surfaced under a dishonest `count`. Closed across all 3 `list_for_subject` call sites with the `_ENUMERATE_TIERS` filter.
- **Governance precedence** (T5 export op) — the `object_store is None` wiring check ran before the gate, so a sub-agent on an unwired deployment got a `NotImplementedError` instead of `memory_subagent_durable_access_refused`. Reordered gate-before-wiring.
- **Required non-empty selectors** (T5 portal) — `subject_id` / `agent_id` defaulted to `""`, so an empty agent namespace or a `SubjectRef` `ValueError` 500 could slip through; pinned to required `min_length=1` (422 before any core call), both Query + DTO layers.
- **Deployment configurability** (T3) — export bucket + retention were hard-coded; made Settings-driven (bucket shape-validated to the `local_fs` contract; retention 7-year floor, longer allowed).
- **Blank-query fail-closed** (T7) — `query=""` / `"   "` bypassed the `query is None` refusal and ran a vector search on empty text; normalized + refused before search/emit.
- **SDK forwarder mirror** (T7) — `MemorySDK.recall_episodes` didn't forward the new `query`, locking typed-SDK pack authors out of the vector path; mirrored the live signature.
- **Wire-vocab honesty** (T8) — the hand-off must not bless removing `memory_approval_engine_not_available` (a wire-public closed-enum value); reworded to retention-by-default + coordinated-amendment-to-remove.
- **Verify-floor-at-promotion-time** (Z1c) — the gate ran against fresh full-suite coverage IN the promoting commit (all 5 above floor; no repair needed).
