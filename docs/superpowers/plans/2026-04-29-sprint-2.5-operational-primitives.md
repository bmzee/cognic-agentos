## Doctrine-review checklist (resolve BEFORE T1 implementation begins)

The author flagged these architectural decisions as needing user review against ADRs / BUILD_PLAN before they're locked in by code:

1. **Should `core/sla` emit `decision_history` records on breach detection?** *Plan default: NO* — SLA is pure timer math. Caller (escalation, LLM gateway, etc.) decides whether breach triggers an emission. Keeps SLA pure-functional + trivially testable. Alternative: SLA emits `decision_history` records with `decision_type="sla.breach"` directly, automating the audit trail at the cost of coupling.
2. **Where do escalation transitions land — `decision_history` or a dedicated `escalation_event` chain?** *Plan default: `decision_history`* — escalation IS a governance-relevant decision; reusing the existing chain avoids a new evidence table + new chain-head row + new alembic migration. Read-side projection is O(n) walk; flagged for Sprint 5+ snapshot-table optimisation when escalation volume is real.
3. **Where do guardrail trips land — `audit_event` or a dedicated `guardrail_event` chain?** *Plan default: `audit_event`* — ISO 42001 evidence chain wants "system blocked PII export" as an audit event. Volume concern: every blocked input/output hits the chain-head FOR UPDATE row. Sprint-2's 50-concurrent canary proves the primitive scales; flag for Sprint 5+ if guardrail traffic becomes a hotspot.
4. **Should guardrail passes also emit?** *Plan default: NO* — passes are noise. Only trips emit. ISO 42001 cares about what was blocked.
5. **Failure model for emission inside a guardrail/escalation operation.** *Plan default: fail-loud, mirroring `AuditStore.append`* — caller decides whether the request path can survive without the emission. No silent buffering, no in-process queue.
6. **Strict state-machine for escalation.** *Plan default: open → acknowledged → assigned → (resolved | re-escalated → acknowledged); illegal transitions raise.* Validation reads the current chain end-state under the same compare-and-set discipline Sprint 2 established.

If the user accepts these defaults the plan proceeds as written; otherwise patch the affected tasks before implementation.

```python
# src/cognic_agentos/core/sla.py
from __future__ import annotations

import dataclasses
from datetime import datetime, timedelta
from enum import StrEnum


class SLAStatus(StrEnum):
    GREEN = "green"        # now < warning_at
    WARNING = "warning"    # warning_at <= now < deadline
    BREACHED = "breached"  # now >= deadline


@dataclasses.dataclass(frozen=True, slots=True)
class SLAPolicy:
    """Caller-side policy bundle. Frozen + slotted; safe to share."""
    name: str
    total_budget: timedelta
    warning_threshold: timedelta  # MUST be < total_budget; constructor validates.

    def __post_init__(self) -> None:
        if self.warning_threshold >= self.total_budget:
            raise ValueError(
                f"warning_threshold ({self.warning_threshold}) must be strictly "
                f"less than total_budget ({self.total_budget})"
            )
        if self.total_budget <= timedelta(0):
            raise ValueError(f"total_budget must be positive; got {self.total_budget}")
        if self.warning_threshold < timedelta(0):
            raise ValueError(
                f"warning_threshold must be non-negative; got {self.warning_threshold}"
            )


@dataclasses.dataclass(frozen=True, slots=True)
class SLADeadline:
    """Absolute deadline + warning point for a specific SLA-tracked operation."""
    policy_name: str
    started_at: datetime
    warning_at: datetime
    deadline: datetime


class SLATimer:
    """Pure-functional SLA timer. No DB, no emission, no I/O.

    Two operations:
      - compute_deadline(start, policy): build the absolute SLADeadline.
      - classify(now, deadline): GREEN / WARNING / BREACHED.

    Both methods are static + side-effect-free; the class exists only
    for namespacing.
    """

    @staticmethod
    def compute_deadline(start: datetime, policy: SLAPolicy) -> SLADeadline:
        if start.tzinfo is None or start.tzinfo.utcoffset(start) is None:
            # canonical-form rejects naive datetimes per Sprint 2 R3.
            # Reject here at the boundary too; SLA-tracked timestamps
            # downstream may participate in evidence-pack export.
            raise ValueError(f"start must be timezone-aware; got {start!r}")
        return SLADeadline(
            policy_name=policy.name,
            started_at=start,
            warning_at=start + policy.warning_threshold,
            deadline=start + policy.total_budget,
        )

    @staticmethod
    def classify(now: datetime, deadline: SLADeadline) -> SLAStatus:
        if now.tzinfo is None or now.tzinfo.utcoffset(now) is None:
            raise ValueError(f"now must be timezone-aware; got {now!r}")
        if now >= deadline.deadline:
            return SLAStatus.BREACHED
        if now >= deadline.warning_at:
            return SLAStatus.WARNING
        return SLAStatus.GREEN
```

## Task 2 — `core/escalation`: state-machine + write side

**Critical-controls module — same discipline as Task 1.**

Escalation lifecycle drives a strict state machine; each transition emits exactly one `DecisionRecord` into the existing `decision_history` chain via `DecisionHistoryStore.append`. No new evidence table; no new chain-head row.

State machine:

```
                  ┌─────────────────────────────┐
                  ▼                             │
   open ──► acknowledged ──► assigned ──► resolved
                                   │
                                   └─► re-escalated ──┘
```

Transitions:
- `open` is created by `EscalationStore.open()`; emits `decision_type="escalation.opened"`.
- `acknowledged` ← `EscalationStore.transition(escalation_id, EscalationState.ACKNOWLEDGED, actor_id, reason)`. Valid only from `open` or `re-escalated`.
- `assigned` ← valid only from `acknowledged`.
- `resolved` ← valid only from `assigned`. Terminal.
- `re-escalated` ← valid only from `assigned`. Loops back through `acknowledged`.

Illegal transitions raise `IllegalEscalationTransition`.

- Create: `src/cognic_agentos/core/escalation.py` (state-machine + write side).
- Create: `tests/unit/core/test_escalation.py` (write-side state-machine tests).

```python
# src/cognic_agentos/core/escalation.py (write-side; read-side in T3)
from __future__ import annotations

import dataclasses
import uuid
from datetime import UTC, datetime
from enum import StrEnum
from typing import Any

from sqlalchemy.ext.asyncio import AsyncEngine

from cognic_agentos.core.decision_history import (
    DecisionHistoryStore,
    DecisionRecord,
)


class EscalationState(StrEnum):
    OPEN = "open"
    ACKNOWLEDGED = "acknowledged"
    ASSIGNED = "assigned"
    RESOLVED = "resolved"
    RE_ESCALATED = "re-escalated"


# Adjacency map enforced at runtime. Each key is the FROM state;
# each value is the set of legal TO states.
_LEGAL_TRANSITIONS: dict[EscalationState, frozenset[EscalationState]] = {
    EscalationState.OPEN: frozenset({EscalationState.ACKNOWLEDGED}),
    EscalationState.ACKNOWLEDGED: frozenset({EscalationState.ASSIGNED}),
    EscalationState.ASSIGNED: frozenset({
        EscalationState.RESOLVED,
        EscalationState.RE_ESCALATED,
    }),
    EscalationState.RE_ESCALATED: frozenset({EscalationState.ACKNOWLEDGED}),
    EscalationState.RESOLVED: frozenset(),  # terminal
}


class IllegalEscalationTransition(ValueError):
    """Raised when transition() is called for a transition that is not in
    _LEGAL_TRANSITIONS. Carries from_state + to_state for diagnostics.
    """

    def __init__(
        self,
        escalation_id: uuid.UUID,
        from_state: EscalationState,
        to_state: EscalationState,
    ) -> None:
        super().__init__(
            f"escalation {escalation_id}: illegal transition "
            f"{from_state.value} → {to_state.value}"
        )
        self.escalation_id = escalation_id
        self.from_state = from_state
        self.to_state = to_state


class EscalationStore:
    """Write-side of the escalation state machine. Read-side queries
    (get_by_id / list_open) live in this same module but project from
    the decision_history chain — see Task 3.

    Concurrency: each write goes through DecisionHistoryStore.append,
    which holds the chain-head FOR UPDATE lock for the duration of the
    INSERT + chain-head UPDATE. Two concurrent transitions on the same
    escalation serialise on that lock; the second sees the new chain
    state and validates against it.
    """

    def __init__(self, engine: AsyncEngine) -> None:
        self._engine = engine
        self._history = DecisionHistoryStore(engine)

    async def open(
        self,
        *,
        actor_id: str,
        level: str,
        reason: str,
        request_id: str,
        tenant_id: str | None = None,
    ) -> tuple[uuid.UUID, uuid.UUID, bytes]:
        """Open a new escalation. Returns (escalation_id, decision_record_id, hash).

        Caller-supplied request_id ties the escalation to the originating
        request. level is a free-form string (e.g. "p1", "p2"); pack
        contracts may pin it to an enum later.
        """

        escalation_id = uuid.uuid4()
        record_id, h = await self._history.append(
            DecisionRecord(
                decision_type="escalation.opened",
                request_id=request_id,
                actor_id=actor_id,
                tenant_id=tenant_id,
                payload={
                    "escalation_id": str(escalation_id),
                    "level": level,
                    "reason": reason,
                    "from_state": None,
                    "to_state": EscalationState.OPEN.value,
                },
            )
        )
        return escalation_id, record_id, h

    async def transition(
        self,
        *,
        escalation_id: uuid.UUID,
        actor_id: str,
        new_state: EscalationState,
        reason: str,
        request_id: str,
        tenant_id: str | None = None,
    ) -> tuple[uuid.UUID, bytes]:
        """Transition an existing escalation to a new state.

        Reads the chain to find the escalation's current end state,
        validates the transition is legal, then emits a new
        decision_history record.

        Raises IllegalEscalationTransition on illegal moves.
        Raises EscalationNotFound if escalation_id has no
        ``escalation.opened`` record in the chain.
        """

        current_state = await self._current_state(escalation_id)
        legal_targets = _LEGAL_TRANSITIONS[current_state]
        if new_state not in legal_targets:
            raise IllegalEscalationTransition(
                escalation_id, current_state, new_state
            )

        record_id, h = await self._history.append(
            DecisionRecord(
                decision_type=f"escalation.{new_state.value.replace('-', '_')}",
                request_id=request_id,
                actor_id=actor_id,
                tenant_id=tenant_id,
                payload={
                    "escalation_id": str(escalation_id),
                    "reason": reason,
                    "from_state": current_state.value,
                    "to_state": new_state.value,
                },
            )
        )
        return record_id, h

    async def _current_state(
        self, escalation_id: uuid.UUID
```

### Notes on the dialect-portable filter

The naive shape `WHERE payload->>'escalation_id' = :id` is Postgres-only. Sprint 2's `GovernanceJSON` TypeDecorator stores the column as native JSON on PG/SQLite and CLOB-with-`json.dumps` on Oracle, so a portable JSON-key filter would need dialect branching.

**Sprint 2.5 implementation: server-side filter ONLY on `decision_type LIKE 'escalation.%'`; payload-side filtering happens client-side in Python.** This is the trade-off: an O(n) walk of the escalation-typed slice of the chain. Ten escalations in a 100k-row chain still pulls 10 rows server-side, the rest is fast Python equality. When projection cost matters (Sprint 5+), introduce the snapshot table.

```python
class GuardrailDirection(StrEnum):
    INPUT = "input"
    OUTPUT = "output"


@dataclasses.dataclass(frozen=True, slots=True)
class GuardrailResult:
    """Per-guardrail check result. ``passed=False`` means the
    guardrail caught something; ``matches`` carries diagnostic
    pattern names (NOT raw matched text, to avoid round-tripping
    PII into evidence).
    """

    passed: bool
    matches: tuple[str, ...] = ()  # named patterns matched, not text

    """Aggregate result across the whole pipeline."""
    direction: GuardrailDirection
    passed: bool  # True iff every guardrail in the pipeline passed.
    results: tuple[GuardrailResult, ...]

    name: str
    """Stable identifier; used as the ``guardrail_name`` in

    def check(self, content: str) -> GuardrailResult:
        ...

    short-circuit on first trip. Auditors want the full picture (e.g.
```

```python
import re


class RegexPIIGuardrail:
    """Sprint-2.5 baseline PII filter. Detects rough patterns for:
      - Credit cards (Luhn-light: 13-19 digits, optional separators).
      - US SSN-shape (NNN-NN-NNNN).
      - Phone-number-shape (+? 10-15 digits with optional separators).
      - Email addresses (RFC-5322-light).

    Per BUILD_PLAN: regex-based MVP; ML filters are Wave 2. The
    intentional false-positive bias is correct for Sprint 2.5 — a
    high-recall filter at the perimeter is the right shape for ISO
    42001 evidence; downstream gates can refine.
    """

    name: str = "pii.regex.baseline"

    _PATTERNS: dict[str, re.Pattern[str]] = {
        "credit_card": re.compile(r"\b(?:\d[ -]?){12,18}\d\b"),
        "ssn_us": re.compile(r"\b\d{3}-\d{2}-\d{4}\b"),
        "phone": re.compile(r"(?<!\d)\+?\d{1,3}[ -]?\(?\d{2,4}\)?[ -]?\d{2,4}[ -]?\d{2,9}(?!\d)"),
        "email": re.compile(r"\b[\w.+-]+@[\w-]+\.[\w.-]+\b"),
    }

    def check(self, content: str) -> GuardrailResult:
        matches = tuple(
            sorted(
                pat_name
                for pat_name, pat in self._PATTERNS.items()
                if pat.search(content)
            )
        )
        return GuardrailResult(
            guardrail_name=self.name,
            passed=not matches,
            matches=matches,
            detail=None if not matches else f"{len(matches)} pattern(s) tripped",
        )
```

### Pipeline contract

```python
# completes core/guardrails.py
class GuardrailPipeline:
    # ... existing __init__ ...

    async def check(
        self,
        content: str,
        *,
        direction: GuardrailDirection,
        request_id: str,
        tenant_id: str | None = None,
    ) -> PipelineResult:
        results = tuple(g.check(content) for g in self._guardrails)
        passed = all(r.passed for r in results)

        # Emit one AuditEvent per trip — NOT per pipeline run. Each
        # tripped guardrail gets its own audit row so the chain
        # carries diagnostic granularity. (Alternative: one row with
        # an array of trips; rejected because it muddles the
        # audit-row "what was blocked" question.)
        for r in results:
            if r.passed:
                continue
            await self._audit_store.append(
                AuditEvent(
                    event_type="guardrail.trip",
                    request_id=request_id,
                    tenant_id=tenant_id,
                    payload={
                        "guardrail_name": r.guardrail_name,
                        "direction": direction.value,
                        "matches": list(r.matches),
                        "detail": r.detail,
                    },
                    iso_controls=("ISO42001.A.7.4",),
                )
            )

        return PipelineResult(
            direction=direction,
            passed=passed,
            results=results,
        )
```
