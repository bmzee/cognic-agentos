# Sprint 8.5 — Resumable Session API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `SandboxSession.checkpoint() / suspend()` + `SandboxBackend.wake()` Protocol extensions per ADR-004 §73-93, with a workspace-tar checkpoint mechanism shared by both Wave-1 backends, a `CheckpointStore` orchestrator with per-tenant prefix isolation + retention-floor enforcement + tombstone semantics for destroyed sessions, and cross-backend wire-public parity pinned by two new conformance regressions.

**Architecture:** Sprint 8A + 8B Wave-1 backends gain three additive Protocol methods that route through a NEW `sandbox/checkpoint_store.py` (CC — substantive tenant-isolation + retention boundary) backed by an EXTENDED Sprint-4 `ObjectStoreAdapter.list_prefix()` async-generator method. `destroy()` writes a tombstone sentinel via `CheckpointStore.tombstone_session()`; the wake pipeline checks the tombstone FIRST via `CheckpointStore.load_tombstone()` and fail-closes via `TombstoneCorruptError` on a malformed sentinel. A background reaper enforces the per-tenant retention floor; `core/chain_verifier.py` gains a payload-key walker that proves suspend→wake linkage via `suspend_event_id` + `restored_from_checkpoint_id` (no `decision_history` schema migration). Closed-enum vocabularies grow additively: `SandboxLifecycleEvent` 8→12 + `SandboxRefusalReason` 15→21. NO `CredentialAdapter` Protocol extension in 8.5 — vault-bearing wake is naturally unreachable today via the existing Sprint-8A `sandbox_credential_adapter_not_configured` admission-time refusal; the real `VaultCredentialAdapter` + `mint_lease` / `revoke_lease` extension ship at Sprint 10.

**Tech Stack:** Python 3.12 + existing Sprint-8A `aiodocker` for DockerSibling tar pipe + existing Sprint-8B `kubernetes-asyncio` for KubernetesPod multiplexed-websocket tar pipe + existing Sprint-4 `LocalObjectStoreAdapter` extended with `list_prefix()` async-generator method + existing Sprint-2 `core/canonical.canonical_bytes` for storage-payload serialisation + existing Sprint-2 `core/decision_history.DecisionRecord` (unchanged schema; new payload keys) + existing 8A `sandbox/audit.emit_sandbox_event` (`sandbox/audit.py:65`) as the audit-emission seam reused by 4 new emitter functions.

## Decisions locked at brainstorming + 10-round spec review (2026-05-18)

User-approved across 10 spec review rounds. These are NOT to be re-litigated mid-task. Per the spec patch log, every entry below is a load-bearing design call.

### Q1-Q5 design locks (spec §2)

1. **Q1 workspace-tar mechanism.** Both Wave-1 backends ship the SAME mechanism: `tar` of writable `/workspace` (canonical mount on every sandbox-runtime image per ADR-004 amendment landing via T1) + env metadata + `vault_lease_refs=()` (always empty in 8.5). DockerSibling does NOT use `docker commit`; KubernetesPod does NOT use CRIU. Mid-process state preservation deferred to Wave-2.

2. **Q2 dedicated `sandbox.lifecycle.checkpoint_purged` audit event.** Reaper emits one dedicated chain row per purge; NOT folded into generic `sandbox.lifecycle.destroyed`. Examiners can prove "this checkpoint existed at T1, was purged at T2, retention window was N seconds" without inferring from absence.

3. **Q3 wake-time policy revalidation.** `wake()` reruns the SAME `admit_policy()` Stage-2 pipeline that `create()` runs — against the CURRENT tenant policy / catalog / Rego / settings. Refusal surfaces as `sandbox_wake_policy_revalidation_failed` (wake-time refusal taxonomy, NOT the original create-time reason; the original lives in `detail`).

4. **Q4 fail-loud Vault-lease path NOW — NO `CredentialAdapter` extension in 8.5.** Per the spec §2.4 amendment after source verification at `sandbox/admission.py:96-105` + `tests/unit/sandbox/test_credential_adapter_stub.py:24+99-116`: extending the Protocol now expands Sprint 8.5 scope to include the Sprint-10 lease-API design. The Sprint-8A admission-time refusal `sandbox_credential_adapter_not_configured` (raised at `admission.py` step 3) means no vault-bearing session can be CREATED today, so no vault-bearing wake is reachable. `CredentialAdapter` extension naturally defers to Sprint 10 alongside `VaultCredentialAdapter`. **T5 was DELETED from the task arc** per this lock; T5 does NOT appear in this plan.

5. **Q5 `wake(session_id, *, actor, tenant_id)` Protocol signature.** Identity seam forward-compat for Sprint 10.5 `SchedulerEngine.submit()` wrap per ADR-022.

6. **Extra lock: session_id alone is NEVER authorization.** `wake()` MUST cross-check caller `tenant_id` kwarg against `metadata.tenant_id` and refuse fail-closed via `sandbox_wake_tenant_mismatch` on mismatch — defence-in-depth past the prefix-keyed lookup.

### Tombstone redesign (spec patch round 4-7 — load-bearing)

7. **Tombstone semantics for destroy()** (P1.r4 design call). `destroy()` writes a `<tenant>/<session>/_tombstoned.json` sentinel via the NEW `CheckpointStore.tombstone_session()` method; bytes retained until reaper sweep after the per-tenant retention window. Wake refuses tombstoned sessions fail-closed via the NEW `sandbox_wake_session_tombstoned` closed-enum reason. The Sprint-8A `SandboxSession.destroy(self) -> None` Protocol signature is UNCHANGED (verified at `sandbox/protocol.py:250`); `tombstoned_by` value comes from the session's existing `_actor_subject` field (`docker_sibling.py:677/870/1581/1630/1661`; `kubernetes_pod.py:472/668` symmetric).

8. **`CheckpointStore.load_tombstone()` is a dedicated read seam** (P1.r5). Wake() step 1(a) MUST call `load_tombstone()` BEFORE `load_latest()` — folding tombstone-check into load_latest would erase the order-of-operations.

9. **Malformed tombstone fails CLOSED via `TombstoneCorruptError`** (P1.r6 load-bearing). NEW store-level typed exception declared in `sandbox/checkpoint_store.py` alongside `CheckpointMaxPerSessionRetentionLocked`. Wake() catches `TombstoneCorruptError` and maps to the SAME `sandbox_wake_session_tombstoned` closed-enum value (no new enum value — operator intent "destroyed = MUST NOT wake" survives degradation; tampering signal lives in `detail`).

10. **Cross-backend tombstone-first parity is conformance-test enforced** (P3.r8 + P3.r9). NEW `tests/conformance/sandbox/test_wake_session_tombstoned_conformance.py` parametrises over BOTH backends with 3 cases each (tombstoned refuses / tombstoned+valid-metadata refuses NOT restore / corrupt tombstone refuses via `TombstoneCorruptError`). Env-gated symmetrically on `COGNIC_RUN_DOCKER_SANDBOX=1 AND COGNIC_RUN_K8S_SANDBOX=1`.

### Storage semantics (spec patch round 3 — load-bearing)

11. **Retention floor lives at the REAPER, NOT `ObjectStoreAdapter.retention_seconds`** (P1.r3 design decision). The WORM `retention_seconds` lock at `protocols.py:170-184` makes `delete()` raise during the window — incompatible with max-per-session eviction + explicit destroy paths. `persist()` passes `retention_seconds=None`; reaper reads `metadata.retention_window_s` and refuses to purge inside the window. Bank-grade WORM compliance deferred to Sprint 9 evidence-pack copy.

12. **`CheckpointMaxPerSessionRetentionLocked` typed exception** (P3.r4 + P3.r5). When `persist()` would exceed `max_per_session_cap` AND every existing checkpoint is inside retention, raises typed exception WITHOUT writing the new checkpoint AND WITHOUT emitting any `checkpoint_purged` chain row. NOT a `SandboxRefusalReason` — configuration-tension surface, not a wake-time refusal. NO `retention_window_active` purge_reason invented; 4-value `purge_reason` closed-set stays closed + unchanged.

### Wire-protocol Protocol extensions

13. **`ObjectStoreAdapter.list_prefix()` is an additive Sprint-4 wire-contract change** (P1.1 + P2.r2 + P3.r6). Async-generator shape (plain `def` returning `AsyncIterator[str]`; NOT `async def`). Caller pattern: `async for key in object_store.list_prefix(bucket, prefix): ...`. Sprint-4 `LocalObjectStoreAdapter` implements via `os.scandir()` recursively. T0.5 MUST land before T3 because `CheckpointStore.load_latest()` + `purge_expired()` depend on it.

14. **`CheckpointMetadata` is NOT directly canonical-bytes-safe** (P1.r2 load-bearing). Carries custom dataclass + tuple fields that `core/canonical.canonical_bytes` REJECTS at `canonical.py:38-44,69-70`. Persistence + evidence-pack export go through explicit `to_storage_payload()` / `from_storage_payload()` JSON-native converters. 9-field shape; `vault_lease_refs` is the only defaulted field (= `()`) so it lands last per Python's `@dataclass` ordering rule.

15. **Chain verifier walks suspend→wake via payload keys** (P1.2 + P-followup). `DecisionRecord` schema has only `record_id` / `trace_id` / `span_id` / `langfuse_trace_id` at `core/decision_history.py:188-199` — NO `parent_trace` column. New linkage lives in `payload["suspend_event_id"]` + `payload["restored_from_checkpoint_id"]`; the chain verifier looks up `decision_history WHERE record_id = payload["suspend_event_id"]`. NO `decision_history` schema migration needed.

```python
        self,
        bucket: str,
        prefix: str,
    ) -> AsyncIterator[str]:
```

```python
        Async-generator function (per ObjectStoreAdapter Protocol contract);
        callers use `async for key in adapter.list_prefix(b, p): ...`.

        entries within each directory to guarantee lexicographic order across
        the full yield sequence.

        Path-traversal defence per the existing root-safety doctrine
        (`_validate_and_resolve` at :389; `_get_sync` at :312-321):

        1. Bucket validated via the SAME `_BUCKET_RE` shape check used by
           put/get/delete (delegates to a `_validate_bucket_or_raise` helper
           extracted from `_validate_and_resolve`; NO regex duplication).
        2. Prefix rejected outright if it contains `..` segments, NUL bytes,
           or leading `/` (mirrors the `_KEY_RE` + structural checks in
           `_validate_and_resolve`).
        3. Bucket root path resolved (NOT strict — bucket may not exist yet);
           refused with `PathTraversalError` if the resolved bucket root is
           not relative to `self._root` (symlink escape at bucket level).
        4. Prefix join resolved (NOT strict); refused with `PathTraversalError`
           if the resolved prefix is not relative to the resolved bucket
           root (catches a symlink swap that points the prefix dir outside
           the bucket).
        5. **At every recursion step in the walk**, EACH visited directory
           re-resolved + re-checked `is_relative_to(self._root)` BEFORE
           descending or yielding files. This catches symlinks placed
           anywhere in the directory tree that escape root mid-walk (e.g.,
           an attacker creates `<bucket>/tenant-a/escape -> /etc`; without
           this check the walker would happily yield `/etc/passwd` as a
           bogus "key" under tenant-a's prefix).

           `is_relative_to(self._root)` BEFORE yielding (defence-in-depth
           against a symlinked file at the leaf).

        # _KEY_RE-style structural checks via the new

        # extracted in this same change. NO regex duplication.
        self._validate_bucket_or_raise(bucket)
        self._validate_prefix_or_raise(prefix)

        # Step 3: bucket root resolved + root-safety check.
        bucket_root_target = self._root / bucket
        try:
            bucket_root_resolved = bucket_root_target.resolve()
        except OSError:
            raise PathTraversalError(
                f"failed to resolve bucket root {bucket_root_target!s}"
            ) from None

            raise PathTraversalError(
                f"resolved bucket root {bucket_root_resolved!s} is not under "
                f"root {self._root!s} (symlink escape at bucket level)"
            )

        # Step 4: prefix join resolved + bucket-relativity check.
        prefix_target = bucket_root_resolved / prefix if prefix else bucket_root_resolved
        try:
            prefix_resolved = prefix_target.resolve()
        except OSError:
            # Prefix doesn't exist yet — yield empty iterator (NOT an error).
            return
        if not prefix_resolved.is_relative_to(bucket_root_resolved):
            raise PathTraversalError(
                f"resolved prefix {prefix_resolved!s} is not under bucket root "
                f"{bucket_root_resolved!s} (symlink escape at prefix level)"
            )

        # If the prefix directory does not exist OR is not a directory,
        # yield nothing (normal empty-iterator behavior per Protocol §3.5).
        if not prefix_resolved.exists() or not prefix_resolved.is_dir():
            return

        # Step 5+6: lazy recursive walk with per-step DUAL root-relativity check
        # (root AND prefix_resolved bound — P1.r11 fix for cross-prefix symlink).
        for key in self._walk_keys_root_safe(
            prefix_resolved,
            bucket_root_resolved,
            walk_root_bound=prefix_resolved,
        ):
            yield key

    def _walk_keys_root_safe(
        self,
        prefix_root: Path,
        bucket_root: Path,
        *,
        walk_root_bound: Path,
    ) -> Iterator[str]:
        """Sync recursive sorted walk helper — yields lexicographically
        ordered keys (relative to bucket_root) for every file under
        prefix_root. Sync because Path operations don't benefit from async;
        the outer async-generator yields control to the event loop per loop
        iteration.

        DUAL ROOT-SAFETY INVARIANT (P1.r10 + P1.r11 load-bearing — every
        visited directory AND every yielded file MUST canonicalise under
        BOTH `self._root` AND the `walk_root_bound`. The single
        `is_relative_to(self._root)` check (pre-P1.r11) was insufficient:
        a tenant-a symlink pointing at tenant-b within the SAME root
        would pass the root check + still yield tenant-b keys under
        tenant-a's listing — that's a tenant-isolation breach. The dual
        check catches the cross-prefix case because the tenant-b target
        is not relative to the tenant-a walk_root_bound.

        `walk_root_bound` is `prefix_resolved` from `list_prefix` step 4 —
        the resolved prefix subtree the caller asked to enumerate. EVERY
        yielded key (and every dir we descend into) MUST live within that
        subtree; a symlink whose resolved target escapes the subtree
        (even if it stays within `self._root`) is a tenant-isolation
        breach and fails-closed with PathTraversalError.
        """
        try:
            entries = sorted(os.scandir(prefix_root), key=lambda e: e.name)
        except (FileNotFoundError, NotADirectoryError):
            return

        for entry in entries:
            entry_path = Path(entry.path)
            try:
                entry_resolved = entry_path.resolve()
            except OSError:
                # Symlink target gone OR loop OR permission denied; skip
                # silently (defence: do NOT yield broken/dangling entries
                # as keys; do NOT raise — a sweep over a multi-million-key
                # tenant should not abort on one bad symlink).
                continue

            # Check 1: resolved target under self._root. Catches escape-
            # out-of-root symlinks (P1.r10 — bucket-level + walk-level).
            if not entry_resolved.is_relative_to(self._root):
                raise PathTraversalError(
                    f"walk encountered entry {entry_path!s} that resolves to "
                    f"{entry_resolved!s} outside root {self._root!s} "
                    f"(symlink escape during list_prefix walk)"
                )

            # Check 2: resolved target under the walk_root_bound (i.e.
            # the prefix subtree the caller asked to enumerate). Catches
            # cross-prefix symlinks WITHIN the same root — e.g.,
            # tenant-a/evil-link -> tenant-b/ (P1.r11 fix). Single
            # is_relative_to(self._root) was insufficient because
            # tenant-b IS under root; the tenant-isolation invariant
            # requires the resolved target to ALSO stay within the
            # original prefix subtree.
            if not entry_resolved.is_relative_to(walk_root_bound):
                raise PathTraversalError(
                    f"walk encountered entry {entry_path!s} that resolves to "
                    f"{entry_resolved!s} outside walk_root_bound "
                    f"{walk_root_bound!s} (cross-prefix symlink — "
                    f"tenant-isolation breach)"
                )

            if entry.is_file(follow_symlinks=False) or (
                entry.is_file() and entry_resolved.is_file()
            ):
                rel = entry_resolved.relative_to(bucket_root)
                yield str(rel).replace(os.sep, "/")
            elif entry.is_dir(follow_symlinks=False) or (
                entry.is_dir() and entry_resolved.is_dir()
            ):
                # Descend with the SAME walk_root_bound — the bound
                # NEVER tightens or relaxes during recursion; it stays
                # pinned to the original prefix subtree the caller
                # asked to enumerate.
                yield from self._walk_keys_root_safe(
                    entry_resolved,
                    bucket_root,
                    walk_root_bound=walk_root_bound,
                )

    def _validate_bucket_or_raise(self, bucket: str) -> None:
        """Extracted from `_validate_and_resolve` steps 1-3 (P1.r10 refactor
        — NO regex duplication; same _BUCKET_RE; same NUL byte + type
        checks). Used by both `_validate_and_resolve` (existing path) and
        `list_prefix` (new path).
        """
        if not isinstance(bucket, str):
            raise PathTraversalError(f"bucket must be str; got {type(bucket).__name__}")
        if "\x00" in bucket:
            raise PathTraversalError("NUL byte in bucket — path-traversal-bait rejected")
        if not _BUCKET_RE.match(bucket):
            raise PathTraversalError(f"invalid bucket {bucket!r}: must match {_BUCKET_RE.pattern}")

    def _validate_prefix_or_raise(self, prefix: str) -> None:
        """Prefix-specific structural validator. Prefix has slightly
        different shape than key: it may be empty (= whole bucket walk)
        OR end with `/`; it may contain `/` separators internally; but it
        MUST NOT contain `..` segments, NUL bytes, or leading `/`
        (mirrors `_KEY_RE` semantics).
        """
        if not isinstance(prefix, str):
            raise PathTraversalError(f"prefix must be str; got {type(prefix).__name__}")
        if "\x00" in prefix:
            raise PathTraversalError("NUL byte in prefix — path-traversal-bait rejected")
        if prefix.startswith("/"):
            raise PathTraversalError(f"invalid prefix {prefix!r}: must not start with '/'")
        if any(seg == ".." for seg in prefix.split("/")):
            raise PathTraversalError(
                f"invalid prefix {prefix!r}: contains '..' path-traversal segment"
            )
```

```python
def _validate_checkpoint_id_or_raise(value: object) -> CheckpointId:
    """**Module-level single source of truth for CheckpointId shape
    validation (P2.r11).** Called by:

    - ``CheckpointStore.validate_checkpoint_id_or_raise`` classmethod
      wrapper (every store entry point that accepts a CheckpointId arg).
    - ``CheckpointMetadata.from_storage_payload`` (evidence-boundary
      parser — rejects corrupt metadata carrying an invalid id BEFORE
      it can smuggle past the NewType façade).

    Centralising in a module-level helper means there is ONE shape
    contract; both the store + parser call sites cannot drift. Pre-P2.r11,
    the parser did `CheckpointId(payload["checkpoint_id"])` directly,
    bypassing the validator — a corrupt metadata object carrying
    ``checkpoint_id="not-hex"`` would parse successfully + smuggle an
    invalid CheckpointId into wake()'s pipeline. Now it raises ValueError
    at the parse seam, mapped at wake() step 1(c) to
    sandbox_wake_checkpoint_corrupt per spec §3.2.
    """
    if not isinstance(value, str):
        raise ValueError(
            f"invalid CheckpointId shape: expected str, got {type(value).__name__}"
        )
    if len(value) != 32:
        raise ValueError(
            f"invalid CheckpointId shape: expected 32-char hex, got len={len(value)} ({value!r})"
        )
    try:
        int(value, 16)
    except ValueError as e:
        raise ValueError(
            f"invalid CheckpointId shape: not hex ({value!r})"
        ) from e
    return CheckpointId(value)

        vlr_list = payload["vault_lease_refs"]
        if not isinstance(vlr_list, list):
            raise ValueError(
                f"CheckpointMetadata.vault_lease_refs must be list (got "

            label=payload["label"],
            created_at=created_at,
```

```python
    from cognic_agentos.core.config import Settings
    from cognic_agentos.sandbox.checkpoint_store import CheckpointStore


logger = logging.getLogger(__name__)


class CheckpointReaper:
    def __init__(
        self,
        *,
        checkpoint_store: "CheckpointStore",
        settings: "Settings",
    ) -> None:
        self._store = checkpoint_store
        self._settings = settings

    async def run_once(self) -> int:
        """Single sweep — delegates to CheckpointStore.purge_expired().

            await self.run_once()
            try:

                logger.info("reaper run_forever cancelled — clean shutdown")
                raise
```
