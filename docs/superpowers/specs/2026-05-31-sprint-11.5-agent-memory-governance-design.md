# Sprint 11.5 — Agent Memory Governance (ADR-019) — Design Spec

**Status:** APPROVED for planning (brainstorming complete 2026-05-31)
**Sprint:** 11.5 (Phase 4), inserts between Sprint 11 (sub-agent primitive) and Sprint 12 (eval harness)
**ADRs:** ADR-019 (primary), ADR-017 (data governance), ADR-014 (approval), ADR-015 (policy-as-code), ADR-018 (emergency), ADR-005 (sub-agent), ADR-006 (ISO 42001), ADR-009 (adapters), ADR-020 (UI events)
**Subsystem:** governance kernel — `core/memory/` (new stop-rule), `core/dlp/` (new seed), `core/emergency/` (new seed, 11.5b)

---

## 1 · Goal

Ship `core/memory/` — the **governed memory substrate** every Layer C agent uses, so banks get the same compliance guarantees on what an agent *remembers* that they already have on tool calls and LLM invocations. Three tiers (`scratch` / `task` / `long_term`), **default-deny `long_term`**, per-write data-class + purpose + consent enforcement, chain-linked audit, and a regulator-erasure pathway.

This sprint also delivers the **SOTA "memory block" shape** (Letta/Hermes-style labeled core memory) as a *first-class governed projection*, so a future agent pack can build a Hermes-style `SOUL.md` / `MEMORY.md` / `USER.md` experience on top of the OS — bank-safe (audited, erasable, default-deny) rather than as private, ungoverned agent-owned files.

## 2 · Scope & non-goals

**In scope (the OS substrate):** the governed memory API + blocks, the per-write/per-recall enforcement gate, a real DLP seed, default-deny Rego bundles, chain-linked audit, storage adapters, vector recall, episodic recall, RBAC scopes, SDK wrappers, and (11.5b) the forget/redact/export/regulator-erasure lifecycle + Redis kill-switch + portal/UI surfaces.

**Explicitly NOT in scope** (the OS/pack boundary):
- The Hermes `SOUL/USER/MEMORY` **experience** — persona text, user-profile curation, the human-like feel — is an **agent pack** (`cognic-agent-*`), Layer C. The OS ships only the governed *shapes* a pack projects onto.
- **Prompt assembly / context injection** (loading blocks into a system prompt at session start, prefix-cache discipline) — agent-runtime / harness, deferred.
- **Self-edit memory tool-loop** (agent autonomously read/write/search via tools) — agent-runtime, deferred. Governed self-improvement is mediated through the existing promotion gates (ADR-019 §42–54), never silent self-modification.
- **Snapshot-injection seam** for parent→child memory hand-off — Wave-1.5 (rides with prompt-injection).
- **Production wiring** — the `MemoryAPI` is built and **DI-tested, not harness-injected**. Same posture the user ratified for the 11b sub-agent primitive: *the primitive is closed; the live dispatch wiring is intentionally later.*

## 5 · Domain model

### 5.1 Tiers
| Tier | Lifetime | Cross-session | Child-writable | Default | Gate |
|---|---|---|---|---|---|
| `scratch` | single invocation | no | **yes** | allow | light — session-audited; DLP classifies; no consent/default-deny |
| `task` | session multi-step (or one resumable-sandbox lifetime) | no | no | allow | full per-write, audited per-write |
| `long_term` | cross-session, cross-agent (within tenant) | yes | no | **deny** | full per-write + manifest declaration + tenant Rego allow |

**Cross-tenant memory is forbidden in Wave 1** (ADR-019 §40) — structurally enforced by `tenant_id` scoping on every row and adapter query; cross-tenant is not a refusal path, it is unrepresentable. The within-tenant control surface is **cross-subject** access (§7.2).

### 5.2 `SubjectRef`
```python
@dataclass(frozen=True)
class SubjectRef:
    kind: Literal["human", "agent"]
    id: str                       # non-empty; validated at construction
    @property
    def canonical(self) -> str:   # "human:<id>" | "agent:<id>"
```
- `human:` subjects are data subjects — GDPR/SBP erasure applies. `agent:` subjects (an agent's own persona) have no human data subject; regulator-erasure of an `agent:` persona is a configuration removal, not a right-to-erasure event. This `human`/`agent` discriminator is **load-bearing** for the 11.5b erasure pathway.
- Empty `id` or a missing scope → the gate refuses; **no tenant-wide (unscoped) record can exist**.

### 5.3 Memory blocks (long_term-only)
A block is a **singleton, bounded, labeled `long_term` projection** — exactly one *active* block per `(tenant_id, subject_ref, agent_id, block_kind)`. `upsert_block` edits in place (a new version supersedes the prior; prior is retained in history). `read_block` returns the active version; `list_blocks(subject)` enumerates a subject's active blocks for a future harness to assemble (injection-*ready*, not injected here).

| `block_kind` | OS-neutral meaning | `SubjectRef` | Hermes analogue (pack, later) |
|---|---|---|---|
| `persona` | agent identity / config | `agent:<agent_id>` | `SOUL.md` |
| `user_profile` | subject preference / profile | `human:<subject_id>` | `USER.md` |
| `agent_notes` | curated operational notes / conventions | `human:` or `agent:` | `MEMORY.md` |

Because a block write is a `long_term` write, it runs the **full default-deny gate**: a pack cannot persist a `persona` block unless it declares `long_term_writes_allowed` AND the tenant Rego allows. Blocks carry a **bounded-size** cap (configurable via `Settings`; default mirrors Hermes-style small curated blocks).

### 5.4 Typed vocabularies (closed enums, wire-public)
- `MemoryTier = Literal["scratch","task","long_term"]`
- `BlockKind = Literal["persona","user_profile","agent_notes"]`
- `DataClass`, `Purpose`, `RetentionPolicy` — `core/memory/` owns **local copies** of these closed enums (a deliberate inline mirror of `cli/_governance_vocab.py`); `core/memory/` MUST NOT import `cli/*` at runtime (the architectural arrow runs `cli → core`). The **restricted** subset is `cli/_governance_vocab.RESTRICTED_DATA_CLASSES` = **`{customer_pii, payment_data, credentials, regulator_communication}`** — the build-time vocabulary packs declare, distinct from the `protocol/mcp_capabilities` runtime set (which uses `payment_action`). The memory + DLP local copies are pinned **lockstep** to `cli/_governance_vocab` via a **test-only drift detector** (no runtime cross-import, per `feedback_drift_detector_test_only_no_runtime_import`).
- `MemoryRefusalReason` — the closed-enum Literal carried by the typed `MemoryOperationRefused` exception (§8.5).

## 6 · MemoryAPI

### 6.1 The 10 operations (a/b split applied)
```python
class MemoryAPI:
    # ---- 11.5a public surface (7) ----
    async def remember(key, value, *, tier, data_classes, purpose,
                       retention_window=None, consent_token=None) -> MemoryRecordId
    async def recall(key, *, tier, purpose) -> MemoryHit | None
    async def recall_episodes(subject, *, similarity_threshold, purpose) -> list[Episode]
    async def list_for_subject(subject) -> list[MemoryRecordId]
    async def upsert_block(kind, *, subject, value, data_classes, purpose,
                          retention_window=None, consent_token=None) -> MemoryRecordId   # long_term-only
    async def read_block(kind, *, subject, purpose) -> MemoryHit | None
    async def list_blocks(subject) -> list[BlockRef]

    # ---- 11.5b (3) — ABSENT from the 11.5a public surface (documented gap, not no-op stubs) ----
    # async def forget(record_id, *, reason) -> ForgetReceipt        # reason="regulator_erasure" → immediate purge
    # async def redact(record_id, *, span, reason) -> RedactionReceipt
    # async def export(scope, *, requester, rbac_scope) -> ExportPackId
```
`subject: SubjectRef`. The 11.5a seven = `{remember, recall, recall_episodes, list_for_subject}` (4 ADR-019 core) ∪ `{upsert_block, read_block, list_blocks}` (3 block). 11.5b adds the 3 remaining ADR-019 core ops (`forget`, `redact`, `export`).

### 6.2 Construction-bound identity + injected seams
The `MemoryAPI` is constructed by the harness (never by Layer C) and **binds the caller context**:
```python
MemoryAPI(*, context: MemoryCallerContext, adapter: MemoryAdapter, dlp: DLPScanner,
          consent: ConsentValidator, policy: PolicyEvaluator,
          kill_switch: MemoryKillSwitchInterrogator = _NullMemoryKillSwitchInterrogator(),
          audit: DecisionHistoryStore, settings: Settings)
```
- `MemoryCallerContext` (frozen): `tenant_id`, `agent_id`, `actor: Principal`, `served_subject: SubjectRef`, `is_subagent: bool`, and the resolved manifest declarations (`long_term_writes_allowed`, `cross_subject_recall`, `memory_read_capabilities`, declared `purposes`, declared `data_classes`, `risk_tier`). The gate reads identity, the **served subject**, and declarations **from this bound context, not from call arguments** — a Layer C caller cannot pass a different `tenant_id` / `agent_id` / subject scope.
- **Subject resolution.** `remember(...)` and `recall(...)` take no subject argument — they operate on `context.served_subject` and can never be cross-subject. The explicit-subject ops (`recall_episodes(subject)`, `list_for_subject(subject)`, `upsert_block(..., subject=...)`, `read_block(..., subject=...)`) compare their requested `subject` to `context.served_subject`; a mismatch is **cross-subject** and refuses (`memory_cross_subject_access_refused`) unless the pack declares `cross_subject_recall` AND tenant Rego allows. Subject scope is therefore construction-bound — a caller cannot smuggle a wider subject per call.
- `kill_switch` is a **required seam**: the default `_NullMemoryKillSwitchInterrogator` raises `NotImplementedError` (pointing at the 11.5b Redis impl) on `is_write_frozen()` — a production deployment that forgets to wire the real kill-switch **fails loud on first write**, never silently allows. Tests inject an `_InactiveMemoryKillSwitch` (returns `False`). Consumer-owned-Protocol + fail-loud-sentinel pattern, identical to the scheduler seams (`feedback_consumer_owned_protocol_for_unlanded_dep`).

## 7 · Enforcement gates

Both gates ship in **11.5a** so the write path is *safe at first commit* (every danger-preventing refusal is present even though the lifecycle ops are deferred). First failure short-circuits and raises the typed `MemoryOperationRefused(reason)`; Layer C receives a typed exception, never a silent drop.

### 7.1 Per-write gate (`remember`, `upsert_block`)
```
0. kill-switch       await kill_switch.is_write_frozen(tenant_id) is True  -> memory_write_frozen
                     (unwired _Null sentinel -> NotImplementedError, fail-loud)
1. sub-agent guard   context.is_subagent AND tier in {task, long_term}     -> memory_subagent_durable_access_refused
1b. subject scope    (upsert_block) subject != context.served_subject AND
                     NOT (manifest.cross_subject_recall AND Rego)          -> memory_cross_subject_access_refused
2. long_term admit   tier == long_term AND NOT (manifest.long_term_writes_allowed
                     AND policy "memory.long_term.allow")                   -> memory_long_term_write_denied
3. DLP               verdict = dlp.scan(value); detected_restricted - declared != {}
                                                                            -> memory_dlp_undeclared_restricted_class
4. purpose           purpose not in manifest.purposes                       -> memory_purpose_not_declared
5. consent           restricted in declared classes AND consent_token
                     missing / expired / subject-mismatch                   -> memory_consent_required | memory_consent_invalid
                     (valid -> chain-link a consent-ledger event)
6. retention         retention_window := min(declared, tenant-max-for-class)  (clamp, not refuse)
7. approval-trans.   risk_tier >= customer_data_write AND tier == long_term -> memory_approval_engine_not_available
                     (mechanical pre-13.5 refusal; lifted by Sprint 13.5 approval routing)
8. write + audit     adapter.put(...) ; emit memory.write (digest, not value) chain-linked
```
`scratch` writes skip steps 2/5/6/7 (DLP still classifies at step 3 for hygiene; default-allow; session-audited, not per-write-audited).

### 7.2 Per-recall gate (`recall`, `read_block`, `recall_episodes`, `list_for_subject`)
```
1. sub-agent guard   context.is_subagent AND tier in {task, long_term}     -> memory_subagent_durable_access_refused
2. capability        manifest lacks memory_read.<tier>                      -> memory_recall_capability_missing
3. subject scope     (explicit-subject reads) subject != context.served_subject AND
                     NOT (manifest.cross_subject_recall AND tenant Rego)    -> memory_cross_subject_access_refused
4. purpose matrix    recall purpose incompatible with write purpose
                     per memory_purpose_matrix.rego                         -> memory_purpose_mismatch
5. audit             emit memory.read (hit | miss) chain-linked
```

### 7.3 Sub-agent memory contract (strict)
Reconciles ADR-019 §143 ("default Wave-1 child gets *fresh* memory; inheritance → Wave-1.5"), the user's "frozen snapshots only", and ADR-005's privilege-de-escalation (child authority ⊆ parent):
- **I1 — No ambient inheritance.** A child's memory context starts fresh; it never automatically sees parent durable memory/blocks.
- **I2 — No live durable-store access (enforced, 11.5a).** With `context.is_subagent` true, the write *and* recall gates refuse all `task`/`long_term` access (`memory_subagent_durable_access_refused`). Children get **`scratch` only**.
- **I3 — Parent-mediated transfer (deferred mechanism).** A child may receive only an inert, parent-curated, read-only snapshot passed as ordinary input; the parent's reads are governed + audited at the *parent* boundary. The formal snapshot-injection seam is **Wave-1.5** (rides with prompt-injection). 11.5 ships I1+I2 enforcement + the ADR-019 §143 clarification, not the seam.

### 7.4 Consent model
Classification is **primary-by-declaration** — `remember(..., data_classes=[...])` is caller-declared and drives the consent requirement (step 5). DLP (step 3) is **defense-in-depth**: it re-scans the value and **refuses under-declaration** of restricted classes (`memory_dlp_undeclared_restricted_class`). This keeps the seed's coverage gaps from being load-bearing — the caller is on the hook to declare; DLP is the safety net for high-signal structured PII.

### 7.5 Closed-enum refusal vocabulary (wire-public, drift-pinned)
**11.5a:** `memory_write_frozen` · `memory_subagent_durable_access_refused` · `memory_long_term_write_denied` · `memory_dlp_undeclared_restricted_class` · `memory_purpose_not_declared` · `memory_consent_required` · `memory_consent_invalid` · `memory_approval_engine_not_available` · `memory_recall_capability_missing` · `memory_cross_subject_access_refused` · `memory_purpose_mismatch`.
**11.5b adds:** forget/redact/export/regulator-erasure reasons + `learning_surface_violation`.
These reasons are the contract bank-overlay consumers + reviewer evidence panels read; a count-pinned drift test guards the Literal.

## 8 · DLP seed (Option A)

`core/dlp/scanner.py`:
```python
class DLPScanner(Protocol):
    def scan(self, value) -> DLPVerdict: ...

@dataclass(frozen=True)
class DLPVerdict:
    detected_classes: frozenset[DataClass]
    redaction_spans: tuple[Span, ...]
    confidence: float
```
The seed (`ChecksumRegexGazetteerScanner`) is **real and deterministic, no model deps**:

| Class | Real recognisers (seed) | Deferred → Sprint 13.5 (needs NER) |
|---|---|---|
| `payment_data` | PAN via **Luhn**, IBAN via **mod-97**, SWIFT/BIC (ISO 9362, context-cued) | — |
| `customer_pii` | email, phone (E.164) | **free-text person names**, addresses, **checksummable national IDs** |
| `regulator_communication` | regulator-name **gazetteer** (SBP/SECP/FCA/OCC/ECB…) | fuzzy/contextual org detection |
| cross-cutting | — | custom recogniser plugins, per-tenant allow/deny, post-call DLP on tool outputs |

Docstring states the deferred gap plainly. The `DLPScanner` Protocol makes a Presidio adapter a clean drop-in at 13.5. No new runtime dependency (stdlib `re`, pure-Python checksums, a gazetteer data file).

## 9 · Policy bundles — default-deny (never permissive)

- `memory.rego`: `data.cognic.memory.long_term.allow` (default **`false`**), `…cross_subject.allow` (default `false`), `…restricted_class_write.allow` (default `false`). Tenant override via a local Rego layer is the only way to permit.
- `memory_purpose_matrix.rego`: `data.cognic.memory.recall.purpose_compatible` — deny on purpose mismatch by default.
- Consumed via the Sprint-4 `core/policy/engine.OPAEngine` seed. **Never stub permissive** — a permissive default would silently authorise `long_term` writes the moment 11.5a ships. Sprint 13.5 expands granularity but does not relax defaults.

## 10 · Storage, audit, migration

- **Adapters:** `MemoryAdapter` Protocol + `PostgresMemoryAdapter` (relational; `tenant_id` column; tier columns + JSONB value) + `RedisMemoryAdapter` (`scratch`-only, sub-second TTL; **fail-closed on Redis-unreachable** — raises an infrastructure error, NOT a Postgres fallback; the §"Storage" fallback is deferred to 11.5b once the reaper gives Postgres-scratch TTL cleanup, per the Cut-A rule). Concrete adapters behind the `adapters` extra.
- **Chain-linkage:** memory events are appended via `DecisionHistoryStore.append_with_precondition` (the repo's atomic chain primitive). The **value never enters the chain** — `memory.write` / `memory.read` payloads carry `tier`, `data_classes`, `purpose`, `retention`, `record_id`, `subject_ref`, and a **redacted-value-digest** only. The JSONB value lives in `memory_records`. ISO 42001 control tags `A.7.4` / `A.8.2` / `A.8.5` / `A.10.2` stamp every memory event (ADR-019).
- **Schema (Alembic revision, erasure-ready from day one):** `memory_records(record_id, tenant_id, subject_ref, agent_id, tier, block_kind NULL, key NULL, value JSONB, data_classes, purpose, retention_until, tombstone NULL, redaction_version, sealed_prior_version_ref NULL, vector_ref NULL, created_at)`. The consent ledger rides `decision_history` in 11.5a (per T7's `memory.consent` chain event); a physical consent table is deferred to 11.5b if/when a writer/query surface exists. A row is **either** a general-memory record (`key` set, `block_kind` NULL) **or** a block (`block_kind` set, `key` NULL) — enforced by a CHECK constraint. **Block singleton-upsert is enforced by a partial unique index** on `(tenant_id, subject_ref, agent_id, block_kind) WHERE block_kind IS NOT NULL AND tombstone IS NULL` (so exactly one active block per identity tuple; superseded versions carry a tombstone). The `tombstone` / `redaction_version` / `sealed_prior_version_ref` columns ship in 11.5a even though `forget`/`redact` land in 11.5b. The migration is a **single dialect-portable Alembic revision** (`0005` precedent; `sa.TIMESTAMP(timezone=True)`) — **not** a separate `db/migrations/oracle/` file; the block partial unique index is created per-dialect within that revision (Postgres/SQLite `postgresql_where`/`sqlite_where`; Oracle a function-based unique index whose `CASE`-NULL expressions exclude non-active/non-block rows).
- **Vector:** embeddings via the existing `ollama` / `openai_compat` embedding adapters → `qdrant` (default per ADR-009); data-class metadata co-stored so semantic recall is purpose-filterable at query time.
- **Episodes:** `recall_episodes(subject, *, similarity_threshold, purpose)` is a **view** over governed `long_term` memories joined to `decision_history` outcomes — episodic recall as an API operation, **not a fourth tier** (exit-criterion invariant).
