**Goal:** Land the non-blocking runtime-approval **decision core** ‚Äî `core/approval/` (engine + store + Rego tier‚Üíflow classifier + RBAC scopes + value-free audit) ‚Äî so high-risk tool invocations have a real approval primitive to consult. **No portal API, no consumer-seam cutover** (those are 13.5b); **no quota/kill-switch** (ADR-018 ‚Üí 13.6).

**Phase:** Phase 4. First of three Sprint-13.5 sub-projects: **13.5a engine core (this)** ‚Üí 13.5b portal API + first seam cutover (MCP host) ‚Üí 13.5c fan-out to the remaining seams + coordinated Rego CONVERT amendment.

**ADR:** ADR-014 (runtime tool approval & risk tiers) √ó ADR-015 (policy-as-code ‚Äî the tier‚Üíflow Rego query) √ó ADR-006 (ISO 42001 control mapping). AgentOS-only; value-free chain. ADR-014/015 acceptance = Human-only.

## ¬ß2 Approval state machine + `check()` verdict

Five states. `check()` returns the `ApprovalState` directly (one source of truth); consumers proceed **only** on `granted`.

```
                       ‚îå‚îÄ‚îÄ single flow ‚îÄ‚îÄ‚ñ∂ granted
pending ‚îÄ‚îÄgrant(first)‚îÄ‚î§
                       ‚îî‚îÄ‚îÄ 4-eyes flow ‚îÄ‚îÄ‚ñ∂ awaiting_second ‚îÄ‚îÄgrant_second‚îÄ‚îÄ‚ñ∂ granted
{pending, awaiting_second} ‚îÄ‚îÄdeny‚îÄ‚îÄ‚ñ∂ denied
{pending, awaiting_second} ‚îÄ‚îÄ(TTL elapsed, lazy at next check/grant/deny)‚îÄ‚îÄ‚ñ∂ expired
```

- `ApprovalState = Literal["pending", "awaiting_second", "granted", "denied", "expired"]` (5).
- **Terminal:** `granted` / `denied` / `expired` ‚Äî no transitions out.
- **4-eyes first grant is NEVER executable (P2 pin):** for `require_4_eyes`, `grant` (first) emits `approval.granted_first` and lands the request in **`awaiting_second`** ‚Äî NOT `granted`. Only `grant_second` reaches `granted`. Consumers proceed **only** on `check(...).state == "granted"`; a request sitting in `awaiting_second` (one of two grants recorded) is non-executable. Pinned by a dedicated test (¬ß11).
- **Lazy expiry is authoritative** and fires on `check` / `grant` / `grant_second` / `deny`: the first call after `expires_at` on a non-terminal request transitions it to `expired`, emits `approval.expired` **once**, and (for a mutation) refuses the mutation. **Safety property:** a stale grant cannot resurrect an expired request ‚Äî `grant`/`grant_second` lazy-expire *first*, so a post-TTL grant yields `expired`, never `granted`. No proactive reaper in 13.5a (deferred).
- The pure `validate_transition(*, from_state, action, flow) -> to_state` (in `_types.py`) is the single source of legal pairs; `storage.transition` calls it under the row lock. `action ‚àà {grant_first, grant_second, deny, expire}`.

## ¬ß3 Engine API (`ApprovalEngine`, non-blocking)

```python
def classify(self, *, risk_tier: str) -> ApprovalFlow: ...
    # tools.rego consult (via ApprovalPolicy); auto_run / require_single_approval / require_4_eyes.
    # Pure read; the seam uses this to branch auto-vs-approval (13.5b). No persistence.

async def create_request(self, *, envelope: ApprovalEnvelope) -> ApprovalRequest: ...
    # 1. validate envelope (¬ß7) ‚Äî rejects out-of-vocab risk_tier / data_classes / missing refs / oversize BEFORE any classify or persist.
    # 2. classify(risk_tier); if auto_run -> refuse ApprovalTransitionRefused("auto_tier_no_approval_required").
    # 3. compute envelope_digest = sha256(canonical_bytes(envelope_canonical_form)).
    # 4. persist pending + emit approval.requested (genesis txn). Returns the ApprovalRequest.

async def check(self, *, request_id: uuid.UUID, tenant_id: str) -> ApprovalCheckResult: ...
    # lazy-expire if past expires_at (persist + emit approval.expired once); return the STRUCTURED result
    # ApprovalCheckResult(state, request_id, flow, risk_tier, tool_identity, args_digest, envelope_digest,
    # originator_subject). The display / queue / Sprint-14 path. tenant-scoped: cross-tenant / unknown ->
    # ApprovalRequestNotFound (the 404-equivalent at the store).

async def verify_grant_for_action(self, *, request_id, tenant_id,
                                  expected_args_digest: bytes, expected_tool_identity: str) -> ApprovalCheckResult: ...
    # THE SEAM REPLAY GATE (13.5b consumes THIS, not bare check()): lazy-expire, build the result, then if
    # state == "granted" AND (args_digest != expected_args_digest OR tool_identity != expected_tool_identity)
    # -> raise ApprovalTransitionRefused("approval_binding_mismatch"). A granted request_id therefore cannot be
    # replayed against a different invocation shape ‚Äî the binding is ENGINE-ENFORCED, fail-closed. The seam
    # proceeds ONLY on a returned result.state == "granted" (a clean, binding-matched grant).

# grant / grant_second / deny ALL require approver.actor_type == "human" FIRST (AGENTS.md "Human-only
# decisions" enforced at the CORE boundary, not only the 13.5b portal route) -> else
# ApprovalTransitionRefused("approver_not_human"). The portal RequireHumanActor (13.5b) is then
# DEFENSE-IN-DEPTH, not the sole enforcement boundary.

async def grant(self, *, request_id, tenant_id, approver: ApprovalActor, reason: str | None = None) -> ApprovalState: ...
    # human-only; lazy-expire; RBAC (approver holds the tier's scope); single -> granted, 4-eyes ->
    # awaiting_second (records first_approver); emit approval.granted_first.
    # reason REQUIRED on reason-mandating tiers (customer_data_write / regulator_communication) -> else grant_reason_required.

async def grant_second(self, *, request_id, tenant_id, approver: ApprovalActor, reason: str | None = None) -> ApprovalState: ...
    # human-only; 4-eyes only; requires awaiting_second; distinctness (approver.subject != first_approver != originator);
    # RBAC; reason rule (as grant); -> granted; emit approval.granted_second.

async def deny(self, *, request_id, tenant_id, approver: ApprovalActor, reason: str) -> ApprovalState: ...
    # human-only; RBAC; {pending, awaiting_second} -> denied; emit approval.denied. reason REQUIRED.
```

`ApprovalActor` is a frozen `(subject, tenant_id, scopes: frozenset[str], actor_type)` projection (the engine is `core/`, so it owns a minimal actor projection ‚Äî it does NOT import `portal/rbac/*`; the portal binds its `Actor` ‚Üí `ApprovalActor` at the 13.5b boundary, mirroring `core/vault.VaultLeaseActorRef`). The engine takes an injected `ApprovalPolicy` + `ApprovalRequestStore` + a `clock: Callable[[], datetime]` seam + `Settings` (constructor injection, like `SchedulerEngine`).

## ¬ß4 Store ‚Äî relational `approval_requests` + chain (Doctrine Lock D)

**Mutable row** `approval_requests`: `request_id` (uuid PK) ¬∑ `tenant_id` ¬∑ `flow` ¬∑ `risk_tier` ¬∑ `tool_identity` ¬∑ `originator_subject` ¬∑ `state` ¬∑ `first_approver` / `second_approver` / `denier` (nullable) ¬∑ `envelope_digest` (bytes) ¬∑ **`args_digest` (bytes ‚Äî FIRST-CLASS column, the caller's digest over the raw governed input; the binding key the seam's replay gate compares against per ¬ß3, NOT buried inside `envelope_digest`)** ¬∑ `redacted_context` (bounded text) ¬∑ `data_classes` (GovernanceJSON list) ¬∑ `required_refs` (GovernanceJSON dict) ¬∑ `created_at` ¬∑ `expires_at` ¬∑ `updated_at`. Indexed `(tenant_id, state)` for the 13.5b queue query.

**Atomic transitions** through `DecisionHistoryStore.append_with_precondition` (one txn: chain-head `FOR UPDATE` ‚Üí row `SELECT ‚Ä¶ FOR UPDATE` ‚Üí `validate_transition` under the lock ‚Üí `UPDATE` state-cache ‚Üí chain `INSERT` ‚Üí chain-head `UPDATE`). `create_request` is the genesis path (INSERT pending + `approval.requested`); `grant`/`grant_second`/`deny`/lazy-`expire` are transitions. Mirrors `core/scheduler/storage.py` exactly.

**Five value-free chain events** ‚Äî `decision_type ‚àà {approval.requested, approval.granted_first, approval.granted_second, approval.denied, approval.expired}`. Chain-payload-is-evidence-snapshot: each payload carries `request_id` ¬∑ `tenant_id` ¬∑ `flow` ¬∑ `risk_tier` ¬∑ `tool_identity` ¬∑ `state` ¬∑ `envelope_digest` ¬∑ `args_digest` ¬∑ approver subjects ¬∑ bounded `redacted_context` ¬∑ `required_refs`. **No raw args, ever** (the engine never has them ‚Äî only the caller-supplied digests).

`ApprovalRequestNotFound` (missing/cross-tenant) is caller-distinct from `ApprovalTransitionRefused(reason)` (state-machine / RBAC / distinctness / expiry refusals) ‚Äî mirrors `SchedulerTaskNotFound` vs `SchedulerTransitionRefused`.

## ¬ß7 Envelope contract (value-free ‚Äî the locked tightening)

Frozen `ApprovalEnvelope` (caller-supplied, the engine's input contract):

| field | type | notes |
|---|---|---|
| `risk_tier` | str | validated ‚àà the 8-value `RiskTier` vocab (inline mirror ‚Äî `core/` must not import `cli/*`; lockstep drift-tested) |
| `tool_identity` | str | non-empty; pack/tool name |
| `originator_subject` | str | the requesting actor's governance subject |
| `tenant_id` | str | non-empty |
| `data_classes` | tuple[str, ...] | validated ‚àà the data-governance closed-enum (inline mirror) |
| `args_digest` | bytes | caller's hash over the raw governed call input; shape-validated (32 bytes) |
| `redacted_context` | str | caller-redacted, value-free reviewer context; **bounded** (size cap, Settings/constant) |
| `required_refs` | dict[str, str] | e.g. `audit_record_ref`; **`audit_record_ref` REQUIRED when `risk_tier == "regulator_communication"`** (ADR-014 ¬ß35) |

The engine **validates** required-fields-per-flow + size caps + enum membership, then computes its **own** `envelope_digest = sha256(canonical_bytes(<canonical envelope form>))`, persists only the bounded redacted context + refs + subjects + tier/flow/state + **both digests** (`envelope_digest` AND the caller-supplied `args_digest`, the latter persisted as a first-class binding key per ¬ß4 so the ¬ß3 replay gate can compare it). **It never receives or stores raw tool args.** Validation failures ‚Üí `ApprovalEnvelopeInvalid(reason)` (closed-enum, ¬ß10), raised **before** classify/persist.

## ¬ß8 Expiry model (per-flow Settings, lazy)

- Two Settings (`core/config.py`, `gt=0` Field constraints): `approval_single_ttl_s` (default **300**) ¬∑ `approval_four_eyes_ttl_s` (default **60**). Per-tier TTL is deferred (Wave-2).
- `expires_at = created_at + ttl(flow)` computed at `create_request` against the injectable `clock` seam (`require_single_approval ‚Üí single_ttl`; `require_4_eyes ‚Üí four_eyes_ttl`).
- **Lazy, authoritative** (¬ß2): `check`/`grant`/`grant_second`/`deny` transition a past-TTL non-terminal request to `expired` (emit once) before doing their work; a mutation on an expired request is refused (`ApprovalTransitionRefused("approval_expired")`).
- **Tightening pin:** tests MUST assert the TTLs are **Settings-driven** (construct the engine with non-default `Settings`, verify `expires_at` reflects them) ‚Äî never hard-coded in the engine.
