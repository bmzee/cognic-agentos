# Sprint 13.8 — MCP Host Production Construction — Design

**ADR:** ADR-002 (MCP plugin protocol). **Status:** design locked, ready for plan.
**Date:** 2026-06-13. **Predecessor:** Sprint 13.7 (scheduler composition-root wiring).

## 1. Goal + posture

Production-construct the `MCPHost` at the composition root so a deployed AgentOS
reaches a real, approval-wired MCP host on `app.state.mcp_host` — executing the
long-deferred **"Sprint-5 T9"** the existing code anticipates (the transport ctor
docstring names "T9's `MCPHost.call_tool`" as its consumer; `app.py` carries the
mount placeholder). Every protocol piece already exists; 13.8 builds **only the
construction wiring + the manifest→`MCPServerEntry` mapper + one read-only
registry accessor**.

**Posture (the 13.7 honesty pattern):** the host is **production-constructed /
composition-root-bound, NOT live-exercised.** No portal route consumes
`app.state.mcp_host` today (grep-confirmed), so the 13.5b2 approval seam becomes
**wired into a constructed host** (any future `call_tool` consults
`runtime.approval_engine` instead of the `tool_approval_engine_not_available`
fallback) but stays **dormant** until a caller route or other consumer invokes
`call_tool`. The MCP-invocation route + the 14A managed-runtime caller are out of
scope.

## 2. Locked forks (brainstorm 2026-06-13)

1. **Construction location** — the FastAPI lifespan, after `build_runtime`,
   gated on `is_mcp_available()`. Builder lives in **`harness/mcp_host.py`** (keeps
   `app.py` thin; protocol modules stay protocol-owned). `build_runtime` stays
   SDK-free + untouched.
2. **Registry source** — reuse the existing `create_app(plugin_registry=...)`
   seam (`app.py:284`); default to `PluginRegistry(audit_store=runtime.audit_store)`.
   **Sub-fork 2a:** consume the **trusted/registered** set only; do **NOT** run
   discovery/trust registration at startup in 13.8.
3. **Failure posture** — SDK unavailable → `app.state.mcp_host = None` + warning;
   no MCP packs → empty host; malformed `[tool.cognic.mcp]` on a trusted pack →
   skip that pack with a structured warning; authz/transport construction failure
   → fail-soft (`app.state.mcp_host = None` + ERROR log; app still boots). No
   `require_mcp_host` setting yet.
4. **Lifecycle** — a dedicated lifespan-owned `httpx.AsyncClient` for MCP authz,
   closed on shutdown. No sharing of the gateway's client.
5. **Test strategy** — normal unit/lifespan tests, no env-gating (the `mcp` SDK is
   present in the uv test venv AND in CI `--all-extras`). The SDK-absent branch is
   tested by monkeypatching `is_mcp_available()`/`require_mcp`.
6. **Scope exclusions** — construction + mount + approval-engine wiring ONLY. No
   MCP-invocation/list route, no 14A managed-runtime caller, no sandbox backend,
   no workflow work.
7. **Mapper source (the one CC touch)** — add a **read-only, additive**
   `PluginRegistry` accessor; derive `package_name` from the stored entry-point
   value **without** `EntryPoint.load()` / pack-code execution. NO
   distribution→package normalization (quiet-correctness-bug risk on non-matching
   wheel layouts).

## 3. Architecture

```
FastAPI lifespan (portal/api/app.py)
  build_runtime(settings, adapters)  ->  runtime           [SDK-free, untouched]
  app.state.scheduler = runtime.scheduler                  [13.7]
  IF is_mcp_available():                                    [SDK gate, 13.8]
      mcp_http_client = httpx.AsyncClient(...)             [lifespan-owned]
      app.state.mcp_host = build_mcp_host(                 [harness/mcp_host.py]
          registry=<injected OR default PluginRegistry>,
          runtime=runtime, settings=settings,
          http_client=mcp_http_client,
          vault_client=adapters.secret)
  ELSE:
      app.state.mcp_host = None  + structured warning
  ...
  finally:
      await mcp_http_client.aclose()   (if constructed)
      await runtime.aclose(); await adapters.close_all()
```

Three new/changed units, with clear boundaries:

- **`harness/mcp_host.py` (NEW, off-gate composition):** `build_mcp_host(...)`
  assembles authz + transport + the mapped servers + `MCPHost`. SDK-free at
  *import* (the MCP classes import cleanly; `require_mcp()` fires only at
  *construction*, which only runs on the SDK-present lifespan path). Consumes the
  *already-trusted* registry set, so the trust decision stays upstream — off-gate,
  same rationale as the scheduler construction logic in `build_runtime`.
- **`protocol/plugin_registry.py` (CC stop-rule, MODIFIED):** one read-only
  additive accessor exposing the per-registered-pack identity the mapper needs.
- **`portal/api/app.py` (off-gate, MODIFIED):** the SDK-gated lifespan branch +
  `app.state.mcp_host` thread + the lifespan-owned httpx client + close-on-shutdown
  and close-on-failure. The stale Sprint-5 `create_prod_app` placeholder comment is
  superseded (the real home is the lifespan, post-`build_runtime`).

## 4. The read-only `PluginRegistry` accessor (the CC touch)

Add `iter_registered_pack_candidates()` to `PluginRegistry` (the accessor yields
registered *candidates*; the mapper decides MCP intent). Contract:

- **Read-only + additive.** Walks the existing `self._records` dict; mutates
  nothing; does not change `known_packs()` / `register()` / `load()` behavior.
- **No pack-code execution.** Derives `package_name` from the stored
  `_RegistryEntry.entry_point.value` (the `dist.module:obj` string — top-level
  module segment) WITHOUT calling `EntryPoint.load()` or importing the package.
  This preserves the Sprint-4/5 deferred-load invariant (pack code is never
  executed at discovery/registration/mapping time).
- **Filters to registered candidates only.** Yields entries with
  `outcome.status == "registered"`; **excludes `refused_at_registration`**. It does
  NOT filter on MCP intent — it yields ALL registered candidates and the **mapper**
  decides MCP intent (via the §5 tri-state probe). Locking this split keeps the
  accessor's responsibility narrow (trusted-set membership) and the MCP semantics in
  the off-gate mapper.
- **Exposes** the minimal identity tuple the builder/mapper consume:
  `distribution_name` (from `PluginRecord.distribution_name`), derived
  `package_name`, and `signature_digest` (from
  `RegistrationOutcome.signature_digest`). The result is a small frozen
  `RegisteredPackCandidate` dataclass (`distribution_name`, `package_name`,
  `signature_digest`). **The accessor does NOT carry the
  `[tool.cognic.mcp]` block:** under Fork 2a no startup registration captured it,
  so the mapper always re-extracts the manifest via `extract_pack_manifest` (§5).
  (Forward-looking only: a future sprint that stores the validated block at
  registration could let the accessor carry it and skip re-extraction — explicitly
  out of 13.8 scope.)

**CC posture:** `protocol/plugin_registry.py` is a critical-controls stop-rule
module on the durable gate (95% line / 90% branch). The accessor is reviewed
(halt-before-commit), additive, and must keep `plugin_registry.py` ≥ floor on
fresh `--cov-branch` data in the touching commit. No CC *promotion* (no new module
to the gate); this is a CC-module *modification*. CC-grade tests required (see §8):
no-load proof, package/distribution-mismatch handling, non-registered
(`refused_at_registration`) exclusion, and a `known_packs()`-unchanged regression.
(MCP-intent filtering is the mapper's concern — NOT the accessor's — so non-MCP
exclusion is a mapper test, §8, not an accessor test.)

## 5. The mapper (SDK-free, in `harness/mcp_host.py`)

`_map_registered_packs_to_servers(registry) -> dict[str, MCPServerEntry]`:

- **Manifest extraction — split the two `MCPManifestError` subclasses.** Call
  `extract_pack_manifest(distribution_name=, package_name=)` (no pack-code import —
  reads `<package_name>/cognic-pack-manifest.toml` via the distribution's RECORD
  index). Both failure subclasses inherit `MCPManifestError`, so they MUST be caught
  SEPARATELY per registry doctrine (`mcp_manifest.py:129,161`) — catching the base
  broadly is the trap:
    - `PackManifestNotFoundError` (manifest file absent) → **no MCP intent → silent
      skip / zero warnings** (Sprint-4-style or non-MCP pack — the registry treats a
      missing manifest as "proceed").
    - `PackManifestMalformedError` (TOML invalid / structurally bad) → **skip +
      structured warning**.
    - success → the parsed manifest dict → tri-state probe.
- **Tri-state block probe (mapper-local — required).** On the parsed manifest, the
  mapper MUST distinguish THREE states of `[tool.cognic.mcp]`, which the existing
  `mcp_capabilities._mcp_block` accessor CANNOT (it collapses both "absent" and
  "present-but-non-dict" into `{}`, `mcp_capabilities.py:352`). The mapper walks
  `tool` → `cognic` → `mcp` locally (each level dict-checked):
    - **absent** (`mcp` key missing under a present `tool.cognic`, or any parent
      absent) → **not MCP-intent → silent skip / zero warnings**.
    - **present-but-malformed** (`mcp` exists but is not a dict, OR required fields
      missing/invalid) → **skip + structured warning** (Fork 3).
    - **present-and-well-shaped** → extract → `MCPServerEntry`.
  So the **silent-skip arm** = {`PackManifestNotFoundError`} ∪ {absent block}; the
  **warning arm** = {`PackManifestMalformedError`} ∪ {present-but-malformed block}.
  The tri-state lives in `harness/mcp_host.py` (off-gate); a public helper in
  `mcp_capabilities` would widen the CC surface beyond the locked registry accessor —
  rejected for 13.8.
- **Field reads are mapper-local — there are NO per-field accessors to reuse.**
  `mcp_capabilities` reads `server_url`/`transport`/`scopes` INLINE inside
  `validate_mcp_manifest` (`mcp.get("transport")`/`("server_url")`/`("scopes")` at
  `:416/452/476`) — there is no reusable `read_transport()`-style helper. So the
  mapper does its OWN field reads + shape checks after the tri-state, reusing only the
  surface that ACTUALLY exists: the constant `_KNOWN_TRANSPORTS` (`:157`),
  `_safe_get_dict` (`:261`), and the data-class helpers `_normalise_data_classes`
  (`:337`) / `_data_classes_shape_violation` (`:281`). Per
  `[[feedback_drift_detector_test_only_no_runtime_import]]`, prefer a mapper-local copy
  of any small constant (the Wave-1 HTTP-family served-set is a subset of
  `_KNOWN_TRANSPORTS` minus `stdio`) pinned by a **drift test** against
  `_KNOWN_TRANSPORTS` over a cross-module private runtime import. Read the risk tier
  from the manifest's risk-tier block.
- **Wave-1 transport family.** A `stdio` transport pack is not served (MCP
  conformance restricts STDIO — admission already refuses it; the mapper emits no
  stdio `MCPServerEntry`).
- **Assemble** `MCPServerEntry(server_id=distribution_name, server_url,
  transport_kind, manifest_scopes=tuple(scopes), risk_tier,
  pack_signature_digest=signature_digest, data_classes=tuple(...))`; key the
  returned dict by `server_id`.

The mapper is the substantive new SDK-free logic and is the primary unit-test
target (§8).

## 6. The builder (`build_mcp_host`)

```python
def build_mcp_host(*, registry, runtime, settings, http_client, vault_client) -> MCPHost:
    authz = MCPAuthzClient(
        settings=settings, vault_client=vault_client, http_client=http_client,
        audit_store=runtime.audit_store, decision_history_store=runtime.decision_history_store,
    )
    transport = StreamableHTTPTransport(authz=authz, settings=settings)  # require_mcp() fires here
    servers = _map_registered_packs_to_servers(registry)
    return MCPHost(
        servers=servers,
        transports={"streamable-http": transport},   # host canonicalizes http/streamable-http
        authz=authz,
        audit_store=runtime.audit_store,
        decision_history_store=runtime.decision_history_store,
        settings=settings,
        approval_engine=runtime.approval_engine,      # 13.5b2 seam WIRED (dormant until a caller)
    )
```

Exact ctor kwargs are grounded: `MCPHost(*, servers, transports, authz,
audit_store, decision_history_store, settings, approval_engine=None)`
(`mcp_host.py:658`); `MCPAuthzClient(*, settings, vault_client, http_client,
audit_store, decision_history_store)` (`mcp_authz.py:377`);
`StreamableHTTPTransport(*, authz, settings, event_hook=None)`
(`mcp_transports.py:163`). `vault_client = adapters.secret` (the bundled Vault
adapter). No new Settings — the authz client + transport read existing MCP
settings (`mcp_oauth_request_timeout_s`, `mcp_call_tool_timeout_s`).

## 7. Failure posture (Fork 3, detailed)

| Case | Behavior |
|---|---|
| `is_mcp_available()` False (kernel image / no SDK) | `app.state.mcp_host = None` + existing structured warning. No httpx client allocated. |
| Zero registered MCP-intent packs | `build_mcp_host` returns `MCPHost` with empty `servers` dict (valid); info-log. |
| Missing manifest (`PackManifestNotFoundError`) OR absent `[tool.cognic.mcp]` block | **Silent skip / zero warnings** — no MCP intent (registry-doctrine "proceed"). |
| Malformed manifest (`PackManifestMalformedError`) OR present-but-malformed `[tool.cognic.mcp]` block | Skip that pack with **a structured warning**; the host builds with the valid packs. |
| authz / transport construction failure (`MCPNotAvailableError` despite the gate, or any exception) | Fail-soft: `app.state.mcp_host = None`, ERROR-level structured log, `await mcp_http_client.aclose()`; the app still boots. |

Trust/signature failures never reach the mapper — under Fork 2a the host consumes
the **registered** set only, so unsigned/untrusted packs (refused upstream) are
absent from the accessor's output.

## 8. Test strategy (Fork 5)

No env-gating (SDK present in test venv + CI). Tiers:

- **Mapper unit tests (SDK-free):** inject fake candidates + manifest dicts (or
  monkeypatch `extract_pack_manifest`) → assert `MCPServerEntry` field mapping. Pin
  the **silent-skip vs warning split as four distinct cases** — two from the
  extraction-exception split, two from the block-presence tri-state:
    - `PackManifestNotFoundError` → silent skip + **zero warnings**;
    - absent `[tool.cognic.mcp]` block (manifest present, no `mcp` key) → silent skip
      + **zero warnings**;
    - `PackManifestMalformedError` → skip + **exactly one structured warning**;
    - present-but-malformed block (non-dict / missing-required-field) → skip +
      **exactly one structured warning**.
  Plus: stdio transport → not served; `signature_digest` carried through; a
  **drift test** that the mapper's transport served-set tracks `_KNOWN_TRANSPORTS`
  (minus `stdio`) and its data-class handling matches the admission helpers.
- **Builder assembly test:** `build_mcp_host(...)` over a stub registry → real
  `MCPAuthzClient` + `StreamableHTTPTransport` + `MCPHost`; **identity pin
  `host._approval_engine is runtime.approval_engine`** (proves the 13.5b2 seam is
  wired) + `host._servers` matches the mapper output. (`require_mcp` is satisfied
  by the present SDK; the kernel-absent path is a separate monkeypatch test.)
- **SDK-absent branch:** monkeypatch `is_mcp_available()`/`require_mcp` → lifespan
  sets `app.state.mcp_host = None` + warning; no httpx client leaked.
- **Lifespan integration:** drive `create_app` lifespan with an injected stub
  registry (fixture MCP candidate) → `app.state.mcp_host` constructed with the
  expected server + closed (httpx client `aclose`d) on shutdown.
- **Registry accessor CC-grade tests (`tests/unit/protocol/...`):** no
  `EntryPoint.load()` is called (assert via a load-spy/monkeypatch); a
  package-name≠distribution-name layout resolves correctly from the entry-point
  value; non-registered (`refused_at_registration`) packs are excluded; existing
  `known_packs()` output is byte-unchanged (regression). Keep
  `plugin_registry.py` ≥ 95/90 on fresh `--cov-branch` data in the same commit.

## 12. Resolved decisions

- Construction in the lifespan (not `build_runtime`, not `create_prod_app`).
- Builder in `harness/mcp_host.py`; mapper SDK-free with a mapper-local tri-state
  block probe + mapper-local field reads (no per-field accessors exist in
  `mcp_capabilities` — only `_KNOWN_TRANSPORTS` + the data-class helpers are reused,
  drift-pinned); `PackManifestNotFoundError`/absent-block → silent skip,
  `PackManifestMalformedError`/present-malformed-block → structured warning.
- Trusted-set source via a new read-only `PluginRegistry` accessor; `package_name`
  derived from the entry-point value, no `load()`, no normalization.
- Consume registered set only; no startup discovery/trust (Fork 2a).
- Fail-soft on construction failure; skip-with-warning per malformed pack.
- Dedicated lifespan-owned httpx client, closed on shutdown + on failure.
- No new Settings; approval seam WIRED but dormant; CC count stays 129;
  `plugin_registry.py` is the one CC-module modification.
