# Sprint 14A-A2 — Portal run route + sandbox approval threading — Design

**ADR:** ADR-022 (scheduler) + ADR-004 (sandbox) + ADR-014 (approval). **Status:** design locked, ready for plan.
**Date:** 2026-06-14. **Predecessor:** 14A-A (executor + DockerSibling backend construction, WIRED-DORMANT).

## 1. Goal + scope

Make the managed-run path **LIVE-exercised** through a production portal caller, and close the 14A-A doc-honesty gap (the sandbox approval-engine branch was UNWIRED). Two-part slice with an internal valve:

- **14A-A2a** — `POST /api/v1/runs`: the production caller. Drives `app.state.managed_run_executor.run(RunRequest)`, LIVE-exercising the executor **and** (via `scheduler.submit`) the **scheduler** approval seam (13.5c2, already wired into the runtime scheduler at 13.7) — for the 14A-A2a `read_only` run shape this means the scheduler approval seam is **consulted on the auto-tier path** (read_only → auto-tier → no pending); scheduler high-risk *pending* is NOT exercised in 14A-A2a (no high-risk run shape is submitted). Includes the executor response-contract extension for pending sandbox approval.
- **14A-A2b** — thread `approval_engine` + `approval_request_id` into both sandbox backends → `admit_policy`, so the same run path LIVE-exercises the **13.5c1 sandbox** approval seam (the run→`202 sandbox_approval_pending`→grant→re-POST contract becomes real).

**Explicitly DEFERRED (out of 14A-A2):** backend-level checkpoint→wake (needs a run-persistence layer + run→session resolver + executor rework — its own slice); MCP `call_tool` exercise (the host is built; it needs an agent-step caller — the agent-runtime workstream 15A/15B); `LocalParentBudgetResolver` + live sub-agent dispatch (the top-level run never sets `parent_task_id`, so the scheduler never consults it — defers with sub-agent dispatch).

## 2. Locked forks (brainstorm 2026-06-14)

1. **F1 — request shape + tenant/actor sourcing.** Body `RunSubmitRequest{pack_id: str, pack_uuid: uuid.UUID, pack_version: str, argv: list[str], approval_request_id: uuid.UUID | None}`. **`tenant_id` + `actor` come ONLY from the bound `Actor`** — the route builds `RunRequest(tenant_id=actor.tenant_id, actor=actor, …)`. The DTO has **no tenant/actor fields** (`extra="forbid"`); `argv` is **non-empty + bounded** (per-element + total length caps — explicit validators). `approval_request_id` is the re-POST carrier (F4).
2. **F2 — RBAC.** New `RunRBACScope = Literal["run.submit"]` family in `portal/rbac/scopes.py`; `RequireScope("run.submit")`; **NO `RequireHumanActor`** — the sandbox approval seam owns the per-tier human checkpoint, so route-level human-gating would double-gate.
3. **F3 — response + status, incl. pending sandbox approval.** Extend `RunResult` with `terminal_state="pending_approval"` + `approval_request_id: str | None`. The executor catches `SandboxLifecycleRefused` whose `.reason` ∈ the `sandbox_approval_*` set at `create()`, **cancels the scheduler task + releases quota** (no session created — same clean path as the queued-cancel), emits value-free `run.pending_approval` evidence, and returns `pending_approval` + the request id — it does **NOT** `scheduler.fail` (pending ≠ failure). Status map: `completed` (incl. non-zero exit) → **200**; `pending_approval` → **202** + `{approval_request_id}`; `refused` (pack/scheduler/sandbox non-approval) → **409** + closed reason; `failed` (infra create/exec) → **502** + reason. Raw `stdout`/`stderr` ARE returned in the 200 body, **base64-encoded with documented size bounds** (so bytes are not an accidental wire ambiguity).
4. **F4 — re-POST / idempotency + threading.** After `PUT /api/v1/approvals/{id}/grant`, the client re-POSTs `/runs` with `approval_request_id`; the executor threads it into `backend.create(…, approval_request_id=…)` → `admit_policy.verify_grant_for_action` re-checks the binding. **Each POST is a fresh run + fresh scheduler task; NO new idempotency layer** — single-use/binding is the approval engine's existing contract.

**Internal valve:** 14A-A2a (route + RBAC + executor response contract incl. `pending_approval`) → 14A-A2b (sandbox `approval_engine` + `approval_request_id` threading through both backends + the `build_sandbox_backend` wiring).

## 3. Grounded facts (the two spec-time checks)

- **`SandboxLifecycleRefused` carrier** (protocol.py:437-449): `__init__(reason: SandboxRefusalReason, *, detail="", approval_request_id: str | None = None)`; exposes `.reason` + `.approval_request_id`. `admission.py:411-417` raises `"sandbox_approval_pending"` *with* `approval_request_id=str(...)`. So the executor's create-failure handler reads `exc.reason` + `exc.approval_request_id`. The `sandbox_approval_*` set (protocol.py): `sandbox_approval_pending` / `_denied` / `_expired` / `_binding_mismatch` / `_request_not_found`.
- **`backend.create` signature** is the `SandboxBackend` Protocol method (protocol.py:650) + `docker_sibling.py:995` + `kubernetes_pod.py:1018`. Adding **`approval_request_id: str | None = None`** is additive/backward-compatible — existing backend tests pass unchanged; the `backend_factory` Literal-arm drift test is unaffected (no new arm). `admit_policy` (admission.py:458) **already** accepts `approval_request_id` — no admission.py contract change; the backends just thread it.

## 4. 14A-A2a — the route (the production caller)

- **`portal/rbac/scopes.py`** (off-gate, wire-protocol-pinned): add `RunRBACScope = Literal["run.submit"]` + `RUN_SCOPES` frozenset; the partition/disjointness test gains the `run.*` family. (Style: plain `= Literal[...]` per the repo convention.)
- **`portal/api/runs/dto.py`** (NEW, off-gate): `RunSubmitRequest` (frozen, `extra="forbid"`: `pack_id`, `pack_uuid`, `pack_version`, `argv: list[str]`, `approval_request_id: uuid.UUID | None = None`; validators: `argv` non-empty + `len(argv) <= _MAX_ARGV_ITEMS` + each item `<= _MAX_ARGV_ITEM_LEN`; **no tenant/actor fields**) + `RunResponse` (frozen: `task_id: str | None`, `terminal_state: Literal["completed","failed","refused","pending_approval"]`, `exit_code: int | None`, `stdout_b64: str`, `stderr_b64: str`, `stdout_bytes: int`, `stderr_bytes: int`, `refusal_reason: str | None`, `approval_request_id: str | None`).
- **`portal/api/runs/routes.py`** (NEW, off-gate): `build_run_routes() -> APIRouter` (closure-factory; `from __future__ import annotations` OMITTED per the FastAPI `Annotated[..., Depends(...)]` invariant). POST `/runs`: `RequireScope("run.submit")` → bound `Actor`; `_require_managed_run_executor(request)` dep → **503 `sandbox_runtime_unavailable`** when `app.state.managed_run_executor is None`; build `RunRequest(tenant_id=actor.tenant_id, pack_id=body.pack_id, pack_uuid=body.pack_uuid, pack_version=body.pack_version, argv=tuple(body.argv), actor=actor, approval_request_id=body.approval_request_id)`; `await executor.run(request)`; map `RunResult.terminal_state` → status (200/202/409/502) + `RunResponse` (base64 the raw bytes). **Mount semantics (corrected):** the router is mounted **unconditionally at construction time** in `create_app` (the router has no `app.state` dependency at mount — `app.state.managed_run_executor` is `None` until the lifespan populates it, so a construction-time mount gate on it would never mount the route). The **request-time** `_require_managed_run_executor(request)` dep is the **sole** gate — it returns **503 `sandbox_runtime_unavailable`** whenever the lifespan did not populate the executor (SDK absent / `sandbox_runtime_enabled=False` / construction fail-soft). The route does **NOT** take `decision_history_store` as a mount dependency — the executor already owns DH; the route never touches the chain directly.
- **`core/run/executor.py`** (CC — the response-contract extension, 14A-A2a part): `RunRequest` gains `approval_request_id: uuid.UUID | None = None` (threaded to `backend.create` in 14A-A2b; harmless until the backend accepts it). `RunResult.terminal_state` Literal gains `"pending_approval"`; `RunResult` gains `approval_request_id: str | None`. New `RunPendingReason`/evidence: the create-failure handler distinguishes a `SandboxLifecycleRefused` with `reason ∈ sandbox_approval_*` → `scheduler.cancel(reason="actor_cancelled")` (release quota; no session) + emit value-free `run.pending_approval` (payload: `task_id`, `approval_reason`, `approval_request_id`) + return `RunResult("pending_approval", …, approval_request_id=exc.approval_request_id)`. All other `create()`/`exec()` exceptions stay `scheduler.fail` → `run.failed`. (14A-A2a lands this branch even though the sandbox still uses `approval_engine=None` until 14A-A2b — with `approval_engine=None`, high-risk tiers hit the pre-13.5 Rego refusal `sandbox_high_risk_tier_refused_pre_13_5` → `failed`, and `read_only` admits; the `pending_approval` branch is reachable only once 14A-A2b wires the engine, but the contract + the cancel/release/evidence path are pinned now with a stub backend that raises a `sandbox_approval_pending` refusal.) **Import-boundary pin (kernel-boot cleanliness — mandatory):** the create-failure handler catches `SandboxLifecycleRefused` (defined in `sandbox/protocol.py`); the `except` clause needs the class at runtime, so it MUST be imported **function-locally inside `run()`** (mirroring the 14A-A `_build_policy` function-local `SandboxPolicy` import) — **NOT** a new module-level sandbox import. `import cognic_agentos.core.run.executor` (module load) must stay hvac-free at kernel boot. T1 keeps the AST no-SDK fence green AND keeps `tests/unit/architecture/test_run_no_sdk_import.py::test_core_run_imports_without_hvac` green — the subprocess probe (hvac blocked via meta-path finder) re-asserts that adding the `SandboxLifecycleRefused` handler introduces **no** module-level `sandbox.*`/`hvac` import. This pin is an explicit T1 step + acceptance check in the plan.

## 5. 14A-A2b — sandbox approval threading (the CC change)

- **`sandbox/protocol.py`** (CC): `SandboxBackend.create` Protocol gains `approval_request_id: str | None = None` (additive).
- **`sandbox/backends/docker_sibling.py`** (CC): `__init__` gains `approval_engine: ApprovalEngine | None = None` + `self._approval_engine`; `create()` gains `approval_request_id: str | None = None`; thread `approval_engine=self._approval_engine, approval_request_id=approval_request_id` into BOTH `admit_policy(...)` call sites (cold-create ~:1110 + wake-revalidation ~:2547).
- **`sandbox/backends/kubernetes_pod.py`** (CC): the SAME change (cross-backend symmetry) at its `__init__` + `create()` + admit_policy call site(s). Behavioural lockstep with docker_sibling.
- **`harness/sandbox.py`** (off-gate): `build_sandbox_backend` threads `approval_engine=runtime.approval_engine` into `get_backend(...)` (the factory forwards `**kwargs`); the lifespan already has `runtime.approval_engine` (built unconditionally at 13.5b1).
- **`core/run/executor.py`** (CC): `run()` threads `request.approval_request_id` (str-cast) into `backend.create(…, approval_request_id=…)`. (`admit_policy` already owns the verify/create-request logic — no admission.py change.)
- **No `admission.py` change** — it already accepts `approval_engine` + `approval_request_id` and owns `_consult_approval_engine`. 14A-A2b only WIRES the existing params.

## 6. Body validation + encoding pins (F1 + F3)

- `argv`: non-empty; `len <= _MAX_ARGV_ITEMS` (e.g. 64); each item `str`, `len <= _MAX_ARGV_ITEM_LEN` (e.g. 4096). Empty/oversized → 422 (Pydantic validation). No shell concatenation (passed verbatim, per 14A-A).
- `RunSubmitRequest` `extra="forbid"` — a `tenant_id`/`actor`/`subject` field in the body → 422 (closes confused-deputy at the wire).
- Raw output: `stdout_b64`/`stderr_b64` = base64 of the raw bytes; `stdout_bytes`/`stderr_bytes` = `len()` (the decoded size). Documented response-size note (the route does not stream; a future cap on returned-output size is a follow-up, flagged not silently truncated).

## 9. Resolved decisions
- 14A-A2 = route (a) + sandbox approval threading (b); internal valve after T1; the 3 deferrals (checkpoint→wake, MCP call_tool, sub-agent/parent-budget) are separate workstreams.
- Body: `{pack_id, pack_uuid, pack_version, argv, approval_request_id?}`; tenant+actor from the bound `Actor` only; `argv` non-empty+bounded; `extra="forbid"`.
- `RunRBACScope="run.submit"` + `RequireScope`; no `RequireHumanActor`.
- `RunResult` gains `pending_approval` + `approval_request_id`; sandbox approval-pending → cancel+release+`run.pending_approval`+**202**; completed→200, refused→409, infra failed→502; raw output base64 + bounded.
- Re-POST with `approval_request_id`; fresh run/task each POST; no idempotency layer; threaded into `backend.create`→`admit_policy` (additive optional param on the Protocol + both backends).
- CC modules changed (executor, protocol, both backends); count stays 130; admission.py unchanged.
