# cognic-agent-example-minimal

Sprint-7A T15 **reference** agent pack — minimal-but-valid; inert by
design. The pack demonstrates the per-kind Wave-1 author lifecycle
for `kind = "agent"`:

```
agentos sign --bundle .   # produces the seven attestations + AgentCard JWS
agentos validate .        # passes once attestations exist on disk
agentos test-harness .    # agents: refused with `harness_unsupported_pack_kind`
agentos verify .
```

The committed reference pack is **static-only** — it does NOT ship
pre-generated attestations or `agent_cards/agent-card.jws`.
`agentos validate .` declares `supply_chain.attestation_paths` +
`identity.agent_card_jws_path` and refuses on a clean checkout until
`sign --bundle` populates them. Run sign first, then validate; this
matches the realistic author flow + the lifecycle test.

The harness refusal is intentional. `cli/test_harness.py` Wave-1
narrows the dispatch table to `frozenset({"tool"})`; agent harness
expansion lands in a follow-up Sprint-7B task. **Sign + verify are
kind-aware** — the AgentCard JWS path is part of the agent-pack
lifecycle and is fully exercised by `agentos sign --bundle` + the
verify trust gate.

For the full author guide read `docs/HOW-TO-WRITE-A-PACK.md` in the
cognic-agentos repo.

## What this pack does

`ExampleMinimalAgent.handle(payload, task=...)` ignores the inbound
A2A envelope and returns `{"text": "ok"}`. The pack ships:

- `cognic-pack-manifest.toml` — Wave-1 manifest, all blocks filled.
- `pyproject.toml` — `[project.entry-points."cognic.agents"]`.
- `src/cognic_agent_example_minimal/{__init__.py,agent.py}` — inert
  `ExampleMinimalAgent` subclass.
- `agent_cards/agent-card.json` — the seed card the AgentCard JWS
  signs. The matching `agent-card.jws` is **regenerated by
  `agentos sign --bundle`** at sign time; it is NOT committed to the
  repo, only generated under `tmp_path` during the lifecycle test
  and inside operator bundles at production sign time.
- `attestations/test-signing/test_signing_key.{private,public}.pem`
  + `NOTE.md` — explicit test-only RSA-2048 keypair (matched halves
  committed via narrow `.gitignore` exception). The `prod` profile
  rejects any `signing_key_path` under `examples/` at startup.

## Why a separate pack vs. the `cli/templates/agent/` scaffold

`cli/templates/agent/` is the Jinja-rendered scaffold consumed by
`agentos init-agent`; it carries `AUTHOR-FILL:` placeholders. This
pack carries no placeholders, so it gates the CI lifecycle directly.

## Why the private key is committed

The keypair is **synthetic + test-only** — see
`attestations/test-signing/NOTE.md` for the full rationale. The same
pattern is in force for the T14 task-local fixture pack at
`tests/fixtures/cli_sign_target_pack/`. Production deployments NEVER
reuse these keys; the `prod` settings profile rejects the path
substring `examples/` at startup.
